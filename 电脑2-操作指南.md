## 🔄 日常操作流程

### 每天必做的三件事

```powershell
# 1️⃣ 开工前：拉取电脑1的改动
cd D:\zhuomian\agent
git pull

# 2️⃣ 改代码...

# 3️⃣ 收工前：保存并推送
cd D:\zhuomian\agent
git add -A
git commit -m "写清楚改了什么"
git push            # 可能会弹 GitHub 登录框
```

> ⚠️ **打死也要记住：先 pull → 再干活 → 最后 add/commit/push**

### 查看状态

```powershell
git status          # 改了哪些文件
git log --oneline   # 最近提交记录
```

## 🔑 push 弹登录怎么办？

本电脑走 HTTPS，首次 push（或凭据过期后）会弹出 GitHub 登录窗口。

### 方案：装 Git Credential Manager（推荐）

```powershell
winget install Git.GitCredentialManager
```

装完后，第一次 push 输入一次账号密码，之后自动记住，不用再输。

## ⚠️ 注意事项

| 规则 | 说明 |
|------|------|
| **先 pull 后干活** | 严禁不 pull 直接改 |
| **别同时改同一文件** | 两台电脑同时改一个文件 = 冲突 |
| **commit 信息写清楚** | `优化 Agent-1 路由` 好过 `update` |
| **.env 不同步** | 每台电脑自己创建 `.env` |
| **HTTPS 认证** | 本电脑 SSH 不可用，走 HTTPS |

## 🐛 常见问题

### push 被拒（rejected）？

电脑1 先推了新内容，你需要先 pull：

```powershell
git pull
git push
```

### 冲突了？

```powershell
git pull
# 手动编辑冲突文件，删掉 <<<<<< ====== >>>>>> 标记
git add -A
git commit -m "解决冲突"
git push
```

### 忘了是 HTTPS 还是 SSH？

```powershell
git remote -v
# 看输出：https://github.com/... → HTTPS
#         git@github.com:...     → SSH
```

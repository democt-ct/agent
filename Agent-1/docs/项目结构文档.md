## 1. 项目概览

**项目名称**：Patient Agent Data Service（患者智能辅助 Agent）

**技术栈**：

| 层次 | 技术 |
|------|------|
| Web 框架 | FastAPI 0.103.2 |
| ASGI 服务器 | Uvicorn 0.24.0 |
| ORM | SQLAlchemy 2.0.25 |
| 数据校验 | Pydantic 2.12.5 |
| 数据库 | PostgreSQL 15 + Redis 7 |
| 向量数据库 | ChromaDB（`data/chroma_knowledge/`） |
| 文本生成 | Ollama + qwen2.5-14b-local（本地） |
| 图片理解 | ModelScope Qwen3.5-397B-A17B（远程） |
| 向量模型 | BAAI/bge-small-zh-v1.5（sentence-transformers） |
| TTS | Kokoro（本地）/ OpenAI Compatible TTS（远程） |
| LLM 编排 | LangChain Core + LangChain OpenAI + LangChain HuggingFace |

**核心能力**：

- 患者主档、病历、就诊记录 CRUD
- 自研模块化 MCP 工具层（5 个核心工具）
- 四层记忆系统：短期工作记忆 → 长期摘要记忆 → 知识记忆 → 事实记忆
- 混合 RAG 检索（向量 + 关键词 + 元数据 + 时效）
- 图文问答 + 语音播报
- Memory Debug 可视化

## 3. app/ 核心应用

### 3.1 `app/main.py` — 应用入口

FastAPI 应用工厂，负责：

- 创建 `FastAPI` 实例（title="Patient Agent Data Service", version="0.3.1"）
- `startup` 事件：自动建表 + 预热知识检索器
- 注册五个路由模块：`router`（数据）、`memory_router`（记忆）、`mcp_router`（MCP/Agent）、`stream_router`（SSE 流式）、`evaluation_router`（质量评估）
- 提供 `/health`、`/`、`/tester` 端点

### 3.2 `app/core/` — 基础设施

| 文件 | 职责 |
|------|------|
| `database.py` | SQLAlchemy 引擎创建（PostgreSQL）、Session 工厂、`get_db` 依赖注入生成器 |
| `redis_client.py` | Redis 客户端（单例模式，含 Mock 降级），会话缓存与短期工作记忆存取 |
| `logging.py` | 结构化日志（支持 JSON / text 格式），请求日志中间件 |

## 4. 数据模型层 (app/models/)

所有模型通过 `app/models/__init__.py` 统一导出，在 `app/main.py` 中通过 `import app.models` 触发注册。

### 4.1 患者数据模型

| 模型 | 表名 | 说明 |
|------|------|------|
| `Patient` | `patients` | 患者主档（姓名、性别、出生日期、手机号、过敏史、地址、医院 ID、活跃状态） |
| `MedicalRecord` | `medical_records` | 病历记录（诊断、ICD 编码、主诉、医嘱、检查结果，关联 patient_id） |
| `VisitRecord` | `visit_records` | 就诊记录（就诊日期、科室、医生、就诊类型，关联 patient_id） |

### 4.2 记忆系统模型

| 模型 | 表名 | 说明 |
|------|------|------|
| `MemoryConversationMessage` | `memory_conversation_messages` | 短期对话消息（按 patient_id + session_id 组织） |
| `MemorySessionBufferMessage` | `memory_session_buffer_messages` | 匿名会话缓冲消息（身份确认前暂存） |
| `MemoryBusinessProfile` | `memory_business_profiles` | 业务画像（从病历/就诊提取的医学摘要） |
| `MemoryConversationProfile` | `memory_conversation_profiles` | 对话画像（从对话中提取的沟通特征） |
| `MemoryUserProfile` | `memory_user_profiles` | 融合画像（业务 + 对话合并） |
| `MemoryKeyEvent` | `memory_key_events` | 关键事件（有长期价值的知识点，支持 canonical_key 幂等合并） |
| `MemoryPreference` | `memory_preferences` | 偏好设置（回答风格、术语级别、风险提醒强度等） |
| `MemoryKnowledgeChunk` | `memory_knowledge_chunks` | 知识块（用于 RAG 检索，含 effective_from / expires_at 版本控制） |

## 5. API 路由层 (app/api/)

- **前缀**：`/api/v1`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/patients` | 创建患者 |
| `GET` | `/api/v1/patients` | 查询患者列表 |
| `GET` | `/api/v1/patients/{patient_id}` | 查询单个患者 |
| `DELETE` | `/api/v1/patients/{patient_id}` | 删除患者 |

| `GET` | `/api/v1/patients/{patient_id}/medical-records` | 查询病历列表 |
| `POST` | `/api/v1/patients/{patient_id}/visits` | 创建就诊记录 |

### 5.2 `memory_routes.py` — 记忆系统接口

- **标签**：`memory`

核心端点：

| 方法 | 路径 | 说明 |

## 6. MCP 工具层 (app/mcp/)

### 6.1 核心模块

| 文件 | 职责 |
|------|------|
| `server.py` | `ModularMCPServer` — 工具注册、列表、调用分发、身份解析 |
| `auth.py` | HMAC-SHA256 身份 token 签发与校验 |
| `config.py` | 从 `app/config/local_settings.py` 加载配置，合并默认值（旧路径 `app/mcp/local_settings.py` 向后兼容） |
| `llm_router.py` | LLM 编排：意图识别 → 计划生成 → 答案生成 → 记忆持久化 |
| `schemas.py` | MCP 数据模型（`MCPToolDefinition`、`MCPToolCallResponse`） |
| `speech.py` | TTS 双引擎（OpenAI Compatible / Kokoro），含降级链 |
| `vision.py` | 视觉 LLM 调用，图片 Base64 → 文字摘要，含模型回退 |

> 本地配置文件 `local_settings.py` / `local_settings.example.py` 已迁移到 `app/config/`，详见 [§ 配置层](#配置层-appconfig)。

### 6.2 注册的 MCP 工具

| 工具名 | 功能 | 身份校验 |
|--------|------|---------|
| `issue_identity_token` | 签发测试用身份 token | 仅测试环境 |
| `verify_identity` | 校验 token 有效性 | 返回身份上下文 |
| `get_medical_records` | 查询患者病历 | patient_id 或 auth_token |
| `get_visit_records` | 查询就诊记录 | patient_id 或 auth_token |
| `get_patient_profile` | 聚合患者完整信息 | patient_id 或 auth_token |

### 6.3 LLM Router 问答流程

```
用户输入 → 身份解析 → 图片分析(可选) → 短期记忆更新
    → 长期记忆构建 → 知识检索(RAG) → 意图识别(+规划)
    → 工具执行(MCP) → LLM 答案生成 → TTS(可选) + 记忆持久化
```

## 附录：关键架构模式

### A. 记忆金字塔

```
        查询频率高 / 容量小
  ↑ 工作记忆（内存，最近 5 轮）
  │ 会话缓冲（匿名，瞬态）
  │ 短期消息（DB，有限保留）
  │ 关键事件（DB，长期，去重合并）
  │ 画像（DB，持续更新摘要）
  ↓ 偏好（DB，几乎不变）
        查询频率低 / 容量大
```

### B. 规则优先 + LLM 兜底

在意图识别、实体提取、身份解析等环节：先尝试确定性规则（快、可解释、零成本），失败时再调用 LLM。

### C. 降级链

嵌入层、视觉层、语音层均实现多级降级：首选方案 → 次选方案 → 本地兜底方案。

### D. 匿名 → 实名过渡

```
匿名会话缓冲（不关联 patient_id）
  → promote（身份确认后迁移到正式表）
  → 正式短期消息（关联 patient_id）
```

### E. 数据流水线

```
原始输入 → 身份解析 → 记忆构建 → 上下文组装 → 工具执行 → 答案生成
```

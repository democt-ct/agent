## 🔄 日常操作流程

```powershell
# 1️⃣ 开工前：拉取电脑2的改动
cd D:\zhuomian\agent
git pull

# 2️⃣ 改代码...

# 3️⃣ 收工前：保存并推送
cd D:\zhuomian\agent
git add -A
git commit -m "写清楚改了什么"
git push            # SSH 免密，直接推
```

> ⚠️ **打死也要记住：先 pull → 再干活 → 最后 add/commit/push**

### 查看状态

```powershell
git status          # 改了哪些文件
git log --oneline   # 最近提交记录
```

## ⚠️ 注意事项

| 规则 | 说明 |
|------|------|
| **先 pull 后干活** | 严禁不 pull 直接改 |
| **别同时改同一文件** | 两台电脑同时改一个文件 = 冲突 |
| **commit 信息写清楚** | `优化 Agent-1 路由` 好过 `update` |
| **.env 不同步** | 每台电脑自己创建 `.env` |
| **SSH 免密** | 本电脑走 SSH，push 不用输密码 |

## 🐛 常见问题

### push 被拒（rejected）？

电脑2 先推了新内容，你需要先 pull：

```powershell
git pull
git push
```

### 冲突了？

```powershell
git pull
# 手动编辑冲突文件，删掉 <<<<<< ====== >>>>>> 标记
git add -A
git commit -m "解决冲突"
git push
```

### SSH 突然连不上了？

```powershell
ssh -T git@github.com    # 测试 SSH 连接
```

如果超时，可能网络波动。等一会再试，或者暂时切 HTTPS：
```powershell
git remote set-url origin https://github.com/democt-ct/agent.git
```

# 前言

## 为什么写这份学习指南？

过去几年，大模型的发展催生了大量 Agent 框架。

几乎每天都会出现新的名词：

* LangGraph
* MCP
* A2A
* AutoGen
* CrewAI
* Mastra
* Deep Research
* Claude Code
* OpenHands
* Cursor

如果只是跟着教程学习，很容易形成一种"知识碎片化"的状态：

```
今天学 LangGraph
明天学 MCP
后天学 A2A
```

最后虽然接触了很多框架，却很难回答下面几个问题：

* Agent 到底是什么？
* 为什么需要 Memory？
* Workflow 和 Multi-Agent 的区别是什么？
* MCP 与 Function Calling 有什么关系？
* 为什么企业越来越重视 Context Engineering？
* 一个真正可落地的 Agent System 应该由哪些模块组成？

因此，这份文档的目标不是学习某个框架，而是建立完整的 **Agent Systems Knowledge Map（知识地图）**。

---

# 学习目标

希望最终能够具备以下能力：

* 理解 Agent 的设计思想，而不是只会调用 API。
* 能独立设计一个 Agent System。
* 能阅读优秀开源项目源码。
* 能分析不同 Agent Framework 的架构。
* 能开发生产级 Agent，而不仅仅是 Demo。

最终目标：

> 从 **Agent Demo Developer** 成长为 **Production Agent Systems Engineer**。

# 如何使用本指南

1. **按推荐阅读顺序**（见下方），从 Part I → Part VI 循序渐进
2. **每个 Chapter 点击 📂 链接**，进入详细文件学习（含代码示例 + 实践任务）
3. **理论与实践交替**：每完成一个 Part，用该 Part 的实践文件做综合练习
4. **预计总时间**：4-5 个月（每周 10-15 小时）

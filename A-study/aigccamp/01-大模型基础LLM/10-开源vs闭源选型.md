# 10-开源 vs 闭源选型决策

> 学完这篇要知道：不是"哪个更强"的问题，是在你的约束下 TCO 最低、风险最小的选择——附带盈亏平衡计算和决策树。

---

## 闭源 API vs 开源自部署：两种生意

| | 闭源 API | 开源自部署 |
|------|------|------|
| 计费 | 按 token | 按 GPU 小时 |
| 初期成本 | 接近 0 | GPU + 工程投入 |
| 调用量 ↑ | 成本线性涨 | 边际成本趋 0 |
| 数据流向 | 上传厂商 | 留自己环境 |
| 模型能力 | 顶级 | 略落后（差距缩小中） |
| 工程运维 | 几乎 0 | 显著（引擎/监控/oncall） |

**粗判断**：日均 QPS < 5 → API；> 50 → 自部署；中间地带 → 算 TCO。

---

## 盈亏平衡粗算

举例：中文客服 Agent，2K input + 500 output，QPS=10

| 方案 | 月成本 |
|------|--------|
| 豆包 API | ~¥67,000 |
| 自部署 Qwen2.5-7B（1×A100） | ~¥40,000-50,000 |

QPS 翻倍：API 直接 ×2 = ¥134K，自部署加一张卡 = ¥75K → **自部署优势随量放大**。

QPS 减半：API ¥34K，自部署还是 ¥40K → **API 在小量时便宜**。

## 六个决策维度

**1. 数据合规**（硬约束）
- 数据不能出公司 → 必须自部署
- 数据不能出国 → 国内 API / 自部署
- 一般敏感 → 闭源 + 脱敏

**2. 调用规模**
- 日 <100K → API
- 日 1M-10M → 看 TCO
- 日 >10M → 自部署

**3. 模型能力要求**
- 复杂推理 → 闭源旗舰
- 代码 Agent → Claude / DeepSeek-V3
- 简单分类 → 开源小模型最便宜

**4. 延迟要求**
- <1s TTFT → 国内 API / 自部署近端
- 异步 → 任意

**5. 团队有 ML 工程师吗**
- 没有 → API（运维你扛不住）
- 有 → 开源发挥威力

**6. License 必查**
- Apache 2.0 / MIT：最友好
- Llama License：7 亿 MAU 限制
- 法务过一遍再上线

## 主流模型路线速查

| 路线 | 代表 | 适合 |
|------|------|------|
| 闭源 API | GPT-4o、Claude、Gemini | 原型快、海外、不差钱 |
| 国内 API | 豆包/通义/文心/DeepSeek | 国内、数据不出国 |
| 开源自部署 | Qwen/DeepSeek/Llama | 数据敏感、大调用量 |
| 混合（Bedrock/Azure） | Claude via AWS、GPT via Azure | 想用闭源但数据不出云 |

## 容易踩的坑

1. **"开源=免费"** —— 不免费，运维和 GPU 是真金白银
2. **自部署=数据安全** —— 还要加加密/权限/审计，不光是部署形态
3. **不看 license** —— Llama 的 7 亿 MAU 限制在某些场景会触发
4. **API 突然 deprecate** —— 生产代码始终有 fallback
5. **选最强模型跑简单任务** —— GPT-4.1 跑分类？Qwen-8B 同样效果便宜 100 倍

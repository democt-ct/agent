# 02-Transformer 架构

> 学完这篇要知道：Transformer 的四个核心组件各自干什么、为什么 LLM 都用它、MHA/MQA/GQA 的本质区别。

---

## 一句话讲清 Transformer

**Transformer = Embedding 查表 + 一堆 Attention 块 + FFN 输出**。它的核心创新是一股脑看完整句话（不像 RNN 一个词一个词读），靠"注意力"算清楚每个词和所有其他词的关系。

---

## 四个核心组件

| 组件 | 干啥的 | 一句话理解 |
|------|--------|-----------|
| **Embedding** | 把 token ID 转成向量 | 字典查表——数字变数学 |
| **Self-Attention** | 算每个词和所有词的关系 | "我在句子里应该关注谁" |
| **FFN（前馈网络）** | 对每个位置的向量做一次非线性变换 | 存知识和做推理 |
| **Add & Norm** | 残差连接 + 层归一化 | 防止梯度消失 + 让训练稳 |

---

## Self-Attention 原理（面试必考）

### Q / K / V 是什么？

每个 token 的向量，乘以三个不同的权重矩阵，得到三个向量：

- **Q（Query）**：我要找什么
- **K（Key）**：我是什么，别人来匹配
- **V（Value）**：匹配后，我给出的信息内容

计算步骤：

```
1. 算相似度： Q @ K^T → 得到一个"注意力分数矩阵"
   比如"我 爱 你"三个词 → 3×3 矩阵，每个格子是"词i关注词j的分数"

2. 缩放：除以 √d（防止梯度太小）
3. softmax：把分数转成0~1的权重
4. 加权求和：权重 × V → 每个位置的输出
```

**一句话**：每个词都会看所有的词，用 Q-K 相似度决定"关注谁"，用 V 决定"从被关注者那儿拿什么信息"。

### 为什么需要 √d 缩放？

如果 Q 和 K 的维度 d 很大，点积的方差会是 d——softmax 后会变成"一个值接近 1，其他全是 0"，梯度消失。除以 √d 让方差回到 1。

### 为什么是 Multi-Head？

单头只能看到一种"关系"（比如语法关系）。多头让每个头关注不同维度——一个头看主谓宾、一个头看修饰关系、一个头看指代。综合起来理解更丰富。

## MHA → MQA → GQA：KV Cache 的进化

这是面试高频题。本质是一个 trade-off：**效果好 vs 显存省**。

| | MHA | MQA | GQA |
|------|-----|-----|-----|
| **每头有独立 K/V 吗** | 各有各的 | 全部共享一组 | 分 G 组，组内共享 |
| **KV Cache 大小** | 1×(最大) | 1/heads(最小) | 1/G(折中) |
| **效果** | 最强 | 略有损 | 几乎无损 |
| **谁在用** | GPT-3、早期 BERT | PaLM、Gemini 旧版 | Llama 2/3、Qwen 2/3、Mistral、GPT-4 |

**GQA 为什么能用？** 因为 K 和 V 不像 Q 那么需要多样性——Q 是"我要查什么"，不同头需要不同的查询角度；K/V 是"我是什么/给你什么"，信息相对统一，共享损失小。

**生产上**：2023 年后所有主流模型用 GQA（通常 G=8），几乎免费省 KV Cache。

---

## FFN：Transformer 的"知识存储"

Self-Attention 负责"理解关系"，FFN 负责"存储知识"——两层线性变换中间夹一个激活函数（通常是 SwiGLU）：

```
FFN(x) = W2 · SiLU(W1 · x) ⊙ (W3 · x)
```

FFN 参数量通常是 attention 的 4-8 倍，是模型参数的大头。有研究表明，FFN 里的权重就是"记忆库"——某个 FFN 神经元可能专门存储"巴黎是法国首都"这种事实。

## 位置编码：Transformer 怎么知道顺序？

Self-Attention 本身**不感知顺序**——"我 爱 你"和"你 爱 我"的权重矩阵可以一样。所以必须给 token 加上位置信息。

### 绝对位置编码（GPT-1/2、BERT）

给每个位置一个固定向量，直接加到 embedding 上。问题：训过的长度（如 512）之外就废了。

### RoPE（旋转位置编码，2023 年后标配）

**核心 idea**：不让 Q 和 K 学位置，而是**把 Q 和 K 旋转一个角度，角度 = 位置 × 基础频率**。两个 token 的相对位置差距决定了它们旋转角度之差，Q·K 自然包含了相对位置信息。

优点：**可以外推到没训过的长度**（训 4K 推理 32K 甚至 128K）。Llama、Qwen、Mistral、DeepSeek 全用 RoPE。

---

## 编码器 vs 解码器

| | Encoder-Only（BERT） | Decoder-Only（GPT） | Encoder-Decoder（T5） |
|------|------|------|------|
| **看上下文方式** | 双向全看 | causal mask（只看左边） | 编码器双向+解码器 causal |
| **擅长** | 分类、NER、embedding | 文本生成 | 翻译、摘要 |
| **今天的地位** | embedding 模型（BGE/E5） | **一切 LLM 的基础** | 几乎被 decoder-only 替代 |

为什么 2024 年全是 decoder-only？因为 **next token prediction 这个训练目标对"通用能力"最有效**——做对所有子任务的前提是理解了一切，而生成下一个词正是一个"需要理解一切"的超集任务。

## 一个完整的 forward pass（直觉版）

```
输入: ["我", "爱", "AI"]
  │
  ├─ 1. Embedding: [词ID] → [向量] + 位置编码（RoPE）
  │    结果: (bs, 3, d_model) 的向量
  │
  ├─ 2. ×N 层 Transformer Block:
  │    ├─ RMSNorm（归一化）
  │    ├─ Attention(Q,K,V) + 残差
  │    ├─ RMSNorm
  │    └─ FFN + 残差  
  │
  └─ 3. LM Head（线性层）：把向量映射回词表 → 下一个词的预测
       结果: (bs, 3, 50257)  → 取最后一位置就是下一个 token 概率
```

---

## 容易踩的坑

1. **默认只看 decoder-only** —— 以为 Transformer = GPT。实际上 encoder-only（embedding 模型）和 BERT 在特定场景仍有用
2. **不关心 RoPE 的 θ 设置** —— θ 太小模型偏重局部信息、太大偏重全局，调参不当外推效果崩
3. **忘了 causal mask** —— LLM 训练必须用，做 embedding 提取时要注意不能加
4. **混淆 MHA/GQA/MQA** —— 面试官问"为什么现在都用 GQA"，答不出"K/V 共享的容忍度比 Q 高"就掉分

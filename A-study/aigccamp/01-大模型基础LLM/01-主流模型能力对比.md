# 01-主流模型能力对比

> 学完这篇要知道：各家模型的特点、适用场景、选型逻辑——不是背排行榜，是理解每家擅长什么。

---

## 为什么这章放最前面

选模型是所有 Agent 开发的第一步。选错了后面推倒重来。这章不列排行榜，按"什么任务该用什么模型"组织，附带一个决策表。

## 模型分层速览

按能力分级，不是按参数大小：

| 级别               | 代表模型                                          | 什么任务够用                         |
| ------------------ | ------------------------------------------------- | ------------------------------------ |
| **旗舰**     | GPT-4o、Claude Opus 4、Gemini 2.5 Pro             | 复杂推理、多步规划、长上下文、多模态 |
| **主力**     | GPT-4.1、Claude Sonnet 4、Qwen3-235B              | 日常 Agent、代码、客服、RAG          |
| **轻量**     | GPT-4.1 Nano、Claude Haiku 4、Qwen3-8B            | 分类、抽取、简单改写、边缘部署       |
| **推理专用** | o3/o4-mini、DeepSeek-R1、Claude Opus 4 推理模式   | 数学、编程竞赛、逻辑推理             |
| **开源旗舰** | DeepSeek-V3、Qwen3-235B、Llama 4、Mistral Large 3 | 私有部署、数据合规、高调用量         |

## 各家特点（一句话+关键点）

### OpenAI（GPT 系列）

**一句话**：综合能力最稳，第三方生态最好，但贵。

- 强项：function calling 最成熟、structured output 原生支持、多模态（GPT-4o）
- 弱项：最新模型 API 价格高、国内访问不便
- 适合：Agent 原型、复杂 tool use、国外业务

### Anthropic（Claude 系列）

**一句话**：代码和长上下文最强，安全性最好，风格细腻。

- 强项：200K context 真实可用、代码生成 SOTA、prompt cache 省钱、MCP 生态
- 弱项：国内用不了、模型版本多命名复杂
- 适合：编程 Agent、长文档分析、追求 output 质量

### Google（Gemini 系列）

**一句话**：多模态原生最强，上下文最长（2M），API 免费额度大方。

- 强项：原生多模态（不是拼接的）、2M context、Google 搜索接地气
- 弱项：API 迭代快（经常改）、社区生态不如 OpenAI
- 适合：多模态应用、需要超长上下文、想省钱

### DeepSeek

**一句话**：开源最强性价比，R1 推理能力惊人，中文顶级。

- 强项：V3 便宜+好用、R1 推理对标 o1、开源可自部署、中文 SOTA
- 弱项：多模态弱、API 稳定性偶尔飘、没有海外合规认证
- 适合：中文 Agent、私有部署、推理任务、预算敏感

### 阿里（Qwen 系列）

**一句话**：中文能力顶级，尺寸最全覆盖（0.5B~235B），开源生态好。

- 强项：中文理解和生成 SOTA、尺寸完整、Qwen-Agent 框架
- 弱项：英文不如 Llama、部分版本 license 限制
- 适合：国内业务、端侧部署（小尺寸）、中文客服

### Meta（Llama 系列）

**一句话**：开源社区最大，生态最丰富，英文强中文一般。

- 强项：社区最大（微调版本无数）、英文能力一流
- 弱项：中文偏弱、license 有 7 亿 MAU 限制
- 适合：英文为主、想用开源但不敢碰小众模型

## 选模型的三个核心问题

1. **数据能出去吗？** → 不能就私有部署（Qwen / DeepSeek / Llama）
2. **一天调多少次？** → 超过百万考虑自部署省钱，低于十万走 API
3. **最需要什么能力？** → 不要为了"可能用到的能力"选最贵模型

---

## 容易踩的坑

1. **只看排行榜选模型** —— LMSYS 榜单测的是"人类偏好"，你的业务是"准确率"或"延迟"，不是一回事
2. **用旗舰模型跑简单分类** —— GPT-4.1 跑情感分类？Qwen3-8B 同效果便宜 100 倍
3. **忽略 API 限流** —— 各家 QPM/TPM 有上限，压测别忘了测限流恢复
4. **模型 deprecate 不关心** —— OpenAI 经常下线旧模型，生产代码要有 fallback
5. **中文场景用 Llama** —— 不是不行，但比 Qwen/DeepSeek 明显差一截

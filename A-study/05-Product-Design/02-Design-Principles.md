# 02 - Design Principles — Agent 产品设计原则

> 学习目标：掌握 Agent 产品的设计原则——不做"技术炫技"，做"用户真正需要的产品"

---

## 1. 核心原则

```
用户的感受              你的技术
─────────────          ────────────
"它懂我"        ←      Context Engineering
"它可靠"        ←      HITL + Fallback
"它不慢"        ←      Async + Cache
"它不贵"        ←      Model Selection + Caching
"出错时有救"    ←      HITL + Undo
```

> 用户不关心你用了什么推理策略。只关心这五件事。

---

## 2. 原则一：先确认，再执行

```
❌ Agent："好的，我已经帮你删除了所有过期数据。"
   → 用户："等等！我没让你删！"

✅ Agent："我发现 50 万条过期记录。删除还是归档？"
   → 用户选择 → 执行
```

实现：见 [02-Agent-Architecture/06-Human-in-the-Loop](../02-Agent-Architecture/06-Human-in-the-Loop.md)

HITL 不是技术补丁，是产品设计决策——**给用户保留最终控制权**。

---

## 3. 原则二：渐进式披露

不要让 Agent 一次输出 5000 字的行程文档。分步展示：

```
❌ 一次全给：
  Agent 生成完整行程 → 用户面对一堵文本墙 → 不想读了

✅ 渐进式：
  Step 1: "我找到了 3 个航班，你看看？"
    → 用户选了 MU5678
  Step 2: "好的。成都这几天天气是……我推荐这 3 家酒店？"
    → 用户选了锦江宾馆
  Step 3: "接下来安排行程。你喜欢人文还是自然？"
    → …逐层展开
```

这是 Agent 独有的优势——可以对话式交互，不是"一次生成完事"。

---

## 4. 原则三：可见的进度

Agent 做长任务时，用户盯着空白屏幕 = 焦虑。

```
❌ 不可见：
  "正在规划中……" → [30秒空白] → 结果

✅ 可见进度：
  🔍 正在搜索成都景点…… ✓（3个）
  ✈️ 正在查询航班…… ⏳
  🏨 等待查询酒店……
  → 用户知道 Agent 在干嘛，愿意等
```

实现：见 [03-Production-Engineering/03-Async-Tasks](../03-Production-Engineering/03-Async-Tasks.md) 中的 progress_callback。

---

## 5. 原则四：优雅降级，而非崩溃

```
❌ 崩溃：
  "抱歉，航班搜索 API 暂时不可用，请稍后再试。"
  → 用户：那我整个行程都做不了了？

✅ 降级：
  "航班 API 暂时不可用，我已用缓存数据 + 网页搜索
   为你找到了以下航班信息（标注了数据时间）。
   你也可以稍后重新搜索获得实时价格。"
  → 用户：虽然不完美，但能用
```

实现：见 [03-Production-Engineering/04-Resilience](../03-Production-Engineering/04-Resilience.md)。

---

## 6. 原则五：可撤销

Agent 的操作必须有"后悔药"。

```
✅ 可撤销：
  "已预订 MU5678 航班。如需取消，点击这里。"
  (保留 24 小时免费取消期)

✅ 有事前确认：
  "即将从你的账户扣除 ¥980。确认支付？"
  [确认支付] [取消]
```

---

## 7. 原则六：明确的能力边界

不要让用户猜 Agent 能做什么、不能做什么。

```
❌ 能力边界不清晰：
  用户："帮我分析一下公司财报"
  Agent：开始胡说八道（因为没给财报数据）
  → 用户以为 Agent 能做，其实不能

✅ 能力边界清晰：
  System Prompt 开头：
  "我是旅游规划助手。我能帮你：
   - 搜索和规划行程
   - 比价机票和酒店
   - 查询天气和景点信息
   
   我不能帮你：
   - 预订支付（会引导你到官网）
   - 处理非旅游相关问题"
```

---

## 8. 产品设计自查清单

启动一个新 Agent 项目前，回答：

```
□ 这个任务真的需要 Agent 吗？（三分类框架）
□ 用户能理解 Agent 在做什么吗？（可见进度）
□ 高风险操作有 HITL 吗？（先确认再执行）
□ 外部依赖有降级方案吗？（优雅降级）
□ Agent 的能力边界用户看得到吗？（明确边界）
□ 用户能撤销 Agent 的操作吗？（可撤销）
□ 第一次体验能不能在 30 秒内获得价值？（渐进式披露）
```

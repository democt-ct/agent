# 01 - Agent vs Workflow — 决策框架

> 学习目标：建立清晰的判断标准——什么任务该用 Agent，什么该用 Workflow，什么根本不需要 LLM

---

## 1. 一个灵魂问题

每个 Agent 项目开始前，先问：

> "这个任务真的需要 Agent 吗？"

90% 的场景，答案是否定的。

---

## 2. 三分类决策框架

```
用户需求
    │
    ▼
┌─────────────────────────────────────┐
│  Q1: 这个任务需要 LLM 的推理能力吗？   │
│     需要理解模糊语言、做判断、生成文本？│
└──────────┬──────────────┬───────────┘
           │ 否           │ 是
           ▼              ▼
    不需要 LLM      ┌──────────────────────────┐
    用传统软件解决    │ Q2: 任务的步骤是固定的吗？    │
                    │ 还是需要根据中间结果动态调整？ │
                    └──────────┬───────────────┘
                               │
                    ┌──────────┴──────────┐
                    │ 固定                │ 动态
                    ▼                     ▼
               Workflow                Agent
```

---

## 3. 什么时候用 Workflow

```
✅ Workflow 适合：
  - 步骤固定且已知
  - 分支条件可以穷举
  - 需要高可靠性、低成本
  - 每个步骤的输出是确定的

示例：订单处理
  ① 验证用户身份 → ② 检查库存 → ③ 计算价格 →
  ④ 发起支付 → ⑤ 更新库存 → ⑥ 发送确认邮件
  → 每一步固定，没有"看情况"
```

### Workflow 中嵌入 LLM

Workflow 不是"没有 LLM"，而是"LLM 只用在需要推理的环节"。

```
订单处理 Workflow + LLM：
  ① 验证用户身份
  ② 检查库存
  ③ LLM：根据用户画像推荐附加商品  ← 只在需要推理时用 LLM
  ④ 计算价格
  ⑤ 发起支付
  ⑥ LLM：生成个性化感谢邮件     ← 只在需要生成文本时用 LLM
```

---

## 4. 什么时候用 Agent

```
✅ Agent 适合：
  - 开放式目标（"帮我规划旅行"而不是"订MU5678"）
  - 步骤无法预先确定
  - 需要根据中间结果调整策略
  - 需要多种工具的灵活组合
  - 错误可容忍（有 HITL 兜底）

❌ Agent 不适合：
  - 确定的、可重复的业务流程
  - 需要 100% 可靠性
  - 成本敏感（Agent 比 Workflow 贵 10-100 倍）
  - 用户无法确认/审核 Agent 的决策
```

---

## 5. 什么时候不需要 LLM

```
❌ 不需要 LLM：
  - 纯计算："25 × 37 = ?"
  - 精确查询："订单 #12345 的状态是什么？"
  - 规则判断："用户年龄 ≥ 18 吗？"
  - 数据格式转换："把 CSV 转成 JSON"
  - 简单的 if/else 逻辑

→ 用传统代码，更快、更便宜、更可靠。
```

---

## 6. 实际案例判断

| 需求 | 方案 | 理由 |
|------|------|------|
| "帮我写一封辞职信" | LLM 直接生成 | 一次推理，无工具 |
| "每天自动发一封天气预报邮件" | Workflow + LLM | 固定流程（取数据→生成→发送），LLM只写邮件正文 |
| "帮我规划一次旅行" | Agent | 开放式、多步、需要多种工具组合 |
| "审批超过1000元的报销单" | 传统软件 | 规则判断，不需要推理 |
| "分析一下公司Q2的销售趋势" | Agent | 需要多步：查数据→分析→生成报告 |
| "自动回复客户常见问题" | Workflow + RAG | 固定流程：检索FAQ→LLM生成回复 |

---

## 7. 决策速查表

```
                     步骤固定          步骤动态
                     ─────────        ─────────
需要推理             Workflow          Agent
(理解/判断/生成)     + LLM

不需要推理           传统软件           不适用
(计算/规则/查询)
```

---

## 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 所有问题都想用 Agent 解决 | 成本暴增，稳定性暴降 | 先用决策框架判断 |
| Workflow 可以解决的用了 Agent | 多花 10 倍钱，结果还不稳定 | 固定步骤 → Workflow |
| 不需要 LLM 的也调 LLM | 浪费钱，还有幻觉风险 | Q1 先判断"需要推理吗" |

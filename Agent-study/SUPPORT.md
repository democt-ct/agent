# 支持

使用与你在 Prompthon Agentic Labs 中所需帮助类型相匹配的支持路径。

## 在打开任何内容之前

先阅读：

- `README.md`
- `CONTRIBUTING.md`
- `contributor-kit/index.mdx`
- `contributor-kit/contribution-workflow.mdx`
- `contributor-kit/github-issue-guide.mdx`

这些文档定义了仓库结构、贡献类型、模板以及审查预期。

## 快速分流

| 需求 | 最合适的第一入口 |
| --- | --- |
| 阅读时遇到困惑，找不到支持说明 | Discord `#general-chats` |
| 想提出手册改进，但形状还很早期 | Discord `#handbook-contributors` |
| 发现具体 bug、损坏链接或事实问题 | GitHub `Bug Or Broken Surface Report` |
| 想提出新的手册内容 | GitHub `Content Proposal` |
| 想提出仓库功能或流程变更 | GitHub `Repository Feature Or Process Proposal` |
| 想贡献一个现有任务 | 查看 `#good-first-issues-feed` 后，在 GitHub issue 中评论认领 |
| 需要 PR review 协调 | 优先使用 PR thread，再用 Discord `#code-review` 做轻量协调 |

从 issue 到 pull request 的完整顺序，请阅读
`contributor-kit/contribution-workflow.mdx`。

## 在以下情况打开 issue

如果你有以下情况，请创建 GitHub issue：

- 具体的 bug 或损坏的路径
- 实验室内容提案
- 源项目提案
- practitioner skill package 提案
- 仓库功能或流程提案

请使用仓库的 issue 表单，这样请求会以适合审查的正确格式进入流程。

## 在以下情况使用社区讨论

请使用 Prompthon IO Discord 进行：

- 在你还不确定某件事是否应成为 issue 之前的早期提问
- 帮助选择正确的贡献面
- 尚未明确范围的一般讨论

阅读问题使用 `#general-chats`，贡献分流使用 `#handbook-contributors`，寻找任务使用
`#good-first-issues-feed`，PR 轻量协调使用 `#code-review`，早期生态讨论使用
`#ai-trends`。

Discord: https://discord.gg/sDE2HhGTg4

## 安全报告

不要就漏洞或敏感配置问题提交公开 issue。请改为遵循 `SECURITY.md`。

## 本 lab 覆盖的内容

- 智能体系统基础与术语
- 可复用的智能体模式和工作流机制
- 上下文、协议、评估和可靠性等系统性议题
- 全球与中文智能体工具链的生态对比
- 面向主要智能体产品类别的故事化案例研究
- 快速变化的 radar 与出版扩展

## 仓库结构

| Path | Purpose |
| --- | --- |
| `foundations/` | 核心概念、术语和心智模型 |
| `patterns/` | 可复用的智能体设计模式和机制 |
| `systems/` | 以生产环境为导向的系统主题，如评估与互操作性 |
| `ecosystem/` | 以主题为先的工具、模型和平台对比 |
| `case-studies/` | 面向主要智能体原型的故事化示例 |
| `radar/` | 快速变化的市场和协议追踪 |
| `reading-paths/` | 面向不同受众的 lab 入口 |
| `publications/` | 面向外部阅读扩展的元数据和结构 |
| `contributor-kit/` | 贡献所需的模板和编辑脚手架 |
| local reference archive | 保留在已发布文档表面之外的导入源材料和规划文档 |

仅在 v1 中代码自然适用的位置保留分区本地示例项目：

- `patterns/examples/`
- `systems/examples/`
- `ecosystem/examples/`
- `case-studies/examples/`

面向贡献者的精选来源映射存放在
`contributor-kit/reference-notes/`。

## 编辑方向

- 先路径，后章节
- 从问题到架构再到权衡的故事化页面
- 以主题为先的生态覆盖，并明确区分全球与中文分区
- `references/` 之外采用仓库原生内容，并进行有源可循的适配

## 参考基础与复用边界

参考材料应当用于整理和规划，而不是作为可发布文案或引用脚手架。

教程型仓库不应继续作为公开引用。页面需要外部来源时，应优先使用官方产品文档、框架仓库、协议规范，以及可运行的解决方案仓库。

如果后续从任何上游材料中改编内容，发布前必须先审查署名和许可约束。

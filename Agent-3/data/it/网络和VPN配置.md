## 1. 公司网络架构

### 1.1 网络区域划分

公司网络按安全等级和使用场景划分为四个逻辑区域，所有区域间通过防火墙和访问控制列表（ACL）进行隔离。具体如下：

| 区域 | 说明 | 接入方式 | 安全级别 | 典型设备 |
|------|------|---------|----------|----------|
| **办公网（内网）** | 总部及各分部办公室，员工日常办公 | 有线（RJ45） / 无线（SSID: Corp-WiFi） | **中** | 台式机、笔记本、打印机、NAS |
| **生产网** | 机房、服务器区域、数据库集群 | 有限专网，**不可**无线接入 | **高** | 服务器、存储设备、核心交换机 |
| **访客网** | 访客、面试人员、临时合作伙伴 | 无线（SSID: Corp-Guest），与内网物理隔离 | **低** | 访客手机、笔记本 |
| **VPN** | 远程办公接入（出差、居家、移动办公） | 软件 VPN 客户端（FortiVPN） | **中** | 员工自备设备（BYOD）或公司配发设备 |

**重要定义：**
- **办公网（内网）**：IP 地址段为 `10.60.0.0/16`（有线）和 `10.61.0.0/16`（无线），默认网关分别为 `10.60.0.1` 和 `10.61.0.1`。内网可访问所有公司内部系统（ERP、财务系统、代码仓库、文件服务器）。
- **生产网**：IP 段为 `172.16.0.0/12`，仅允许通过堡垒机（JumpServer）或专用管理终端访问。**禁止**任何无线设备接入。
- **访客网**：IP 段为 `192.168.100.0/24`，仅提供互联网出口，**禁止**访问内网资源。每个访客设备每日流量上限 500MB。
- **VPN**：通过 FortiGate SSL VPN 接入，分配虚拟 IP `10.62.x.x`。**必须**启用双因素认证（密码 + 手机验证码或硬件令牌）。

### 1.2 网络配置参数

| 参数 | 内网有线 | 无线（Corp-WiFi） | VPN（远程接入） |
|------|---------|------------------|-----------------|
| IP 获取方式 | DHCP 自动分配 | DHCP 自动分配 | DHCP（虚拟池） |
| IP 段 | `10.60.x.x` | `10.61.x.x` | `10.62.x.x` |
| 子网掩码 | `255.255.0.0` | `255.255.0.0` | `255.255.255.0` |
| 默认网关 | `10.60.0.1` | `10.61.0.1` | `10.62.0.1` |
| 首选 DNS | `10.60.0.53` | `10.60.0.53` | `10.60.0.53`（通过 VPN 隧道） |
| 备用 DNS | `114.114.114.114` | `114.114.114.114` | 不适用（由 VPN 网关配置） |

**备注：** 办公区无线（Corp-WiFi）使用 **802.1X 认证**，需输入企业账号和密码。每个用户最多同时连接 **5 台**设备。若设备数量超限，需在 IT 管理平台解绑闲置设备。

## 2. VPN 配置

### 2.1 VPN 适用场景（严格条件）

员工在以下任一情况时**必须**使用公司 VPN 访问内网资源：

- **远程办公**：在家或酒店需要连接公司网络（如接入 ERP、财务系统、代码仓库）。
- **出差访问**：在客户现场或机场等公共场所访问内部文件服务器或邮件系统。
- **受限资源**：访问被锁定为内网 IP 的站点（如 Jenkins、GitLab、Wiki）。
- **安全审计**：需要在内网环境进行运维操作（如登录堡垒机）。

**禁止使用 VPN 的场景：**
- 仅需访问互联网（如查资料、收外部邮件），应直接使用公共互联网或访客网络。
- 在公司内部办公区域（已连接办公网），禁止同时开启 VPN（避免路由冲突）。
- 个人设备未安装公司安全软件（如 EDR）或未注册 MDM，禁止接入 VPN。

### 2.2 VPN 客户端安装（Windows 完整流程）

以下为 Windows 10/11 平台标准安装步骤。**Mac 和 Linux 用户请参考附录 A。**

1. 打开浏览器，访问 `https://vpn.company.com`，下载 **FortiVPN** 客户端（安装包约 50MB，MD5 校验码请查看 IT 门户）。
2. 双击安装包，以管理员身份运行。安装过程中保持默认选项，**不要勾选**“启用自动更新”（由 IT 统一推送）。
3. 安装完成后，点击桌面快捷方式启动客户端。
4. 在连接界面输入：
   - **服务器地址**：`vpn.company.com`
   - **端口**：`443`（SSL）/ `10443`（备用）
   - **身份认证**：选择“双因素认证”
5. 点击“连接”，弹出登录窗口。输入企业 AD 账号（如 `zhangsan@company.com`）和密码。
6. **首次使用必做：**
   - 弹出窗口要求绑定手机号（用于接收验证码）。请确保手机号与 HR 系统一致，否则联系 IT 更新。
   - 绑定成功后，每次连接需输入短信验证码或通过硬件令牌（YubiKey）输入 OTP。
7. 连接成功后，任务栏托盘图标变为绿色。可通过 `ipconfig` 查看虚拟 IP 是否为 `10.62.x.x`。
8. **断开连接**：右键图标 → 断开，或直接退出客户端。

### 2.3 VPN 常见问题（FAQ）

| 问题 | 可能原因 | 解决方法 |
|------|---------|----------|
| **连接超时（30001）** | 本地网络限制（如酒店 Wi-Fi 屏蔽 VPN 端口）、防火墙拦截 | 更换网络（使用手机热点），若仍失败则尝试备用端口 `10443` |
| **提示证书无效（-52）** | FortiVPN 客户端版本过旧；或系统时间不同步 | 更新客户端至最新版（v7.0+）；同步系统时间（`控制面板→日期和时间→自动设置`） |
| **连接后无法访问内网资源** | VPN 路由未正确下发；或目标服务器仅允许特定 IP 段 | 断开重连；检查是否在公司内网同时开启了其他 VPN 代理（如 Shadowsocks、Clash）— **必须关闭** |
| **频繁掉线（每 15 分钟断一次）** | 客户端设置了“空闲超时”；或网络不稳定 | 打开客户端设置 → 高级 → 勾选“保持连接”（Keep Alive）；同时联系网络运营商检查丢包率 |
| **忘记 VPN 账号密码** | 忘记 AD 密码或 VPN 专用密码 | 按《密码和账号管理流程》通过自助重置工具（https://password.company.com）重置；或联系 IT 服务台 |
| **手机接收不到验证码** | 手机号未绑定或绑定错位；短信网关延迟 | 登录 IT 自助门户（https://selfservice.company.com）更新手机号；或使用 Passkey（硬件令牌）备用 |

### 2.4 新增：VPN 使用规范与违规处罚

- **禁止**将 VPN 账号分享给非公司人员，一经发现将按《信息安全制度》处以书面警告或解除劳动合同。
- **禁止**通过 VPN 进行非法下载（如盗版影视、大流量 P2P），信息部将监控流量并使用 DPI 分析。违者限制网络访问 30 天。
- **禁止**在 VPN 连接状态下在内网运行端口扫描、漏洞探测等工具。若发现，立即断网并上报信息安全委员会。

## 3. 无线网络使用

### 3.1 无线网络 SSID 说明

| SSID | 适用对象 | 认证方式 | 频段 | 附加条件 |
|------|---------|---------|------|----------|
| **Corp-WiFi** | 全体员工 | 802.1X（企业账号 + 密码） | 5 GHz | 信号覆盖所有办公区；每个账号限 5 台设备 |
| **Corp-WiFi-2.4G** | 仅限已注册的 IoT 设备（打印机、扫描仪、固定工位 IP 电话） | MAC 地址白名单 | 2.4 GHz | 需提交《新设备网络接入申请单》至 IT，审批后方可配置 |
| **Corp-Guest** | 访客、面试人员、客户 | 临时密码（由前台或面试官生成，有效期 4 小时） | 2.4/5 GHz | 仅限互联网访问，**禁止**扫描或连接内网设备 |

**重要定义：**
- **802.1X 认证**：支持 EAP-PEAP 协议。用户名为公司邮箱前缀（如 `zhangsan`），密码为 AD 密码。连接时需信任服务器证书（颁发机构为 `company-ROOT-CA`）。
- **MAC 白名单**：只允许已被 IT 登记 MAC 地址的设备连接 `Corp-WiFi-2.4G`。若设备更换网卡，需重新申请。
- **临时密码生成**：前台人员通过内网门户（`https://guest.company.com`）为访客生成一次性密码，密码长度 8 位，包含大小写字母和数字。

### 3.2 无线网络连接不上（排查步骤）

**情景：** 员工连接 Corp-WiFi 时提示“认证失败”或“无法获取 IP 地址”。

1. **检查 SSID**：确保连接的是 **Corp-WiFi**（5G），而非 Corp-Guest 或家庭网络。
2. **忘记网络重新连接**：
   - Windows：设置 → 网络和互联网 → WiFi → 管理已知网络 → 选择 Corp-WiFi → 忘记。
   - macOS：系统偏好设置 → 网络 → WiFi → 高级 → 删除 Corp-WiFi 条目。
   - 重新连接时输入账号密码，注意密码为 **AD 密码**（非 VPN 密码）。
3. **校验设备数量**：若提示“设备数量上限”，请登录 IT 自助平台（https://selfservice.company.com）查看已注册设备，移除不再使用的设备（如旧手机、离职同事留下的设备）。
4. **检查系统时间**：802.1X 认证依赖系统时间与 NTP 同步。若时间偏差大于 5 分钟，认证将失败。需手动同步：`设置 → 日期和时间 → 自动设置时间`。
5. **询问同事**：如果周围同事同样无法连接，可能是区域 AP 故障，提交 IT 工单并说明楼栋、楼层、座位号。
6. **提交 IT 工单**：若以上步骤全部无效，请通过 https://it.company.com 提交“无线网络无法连接”工单，提供 MAC 地址、错误截图。

### 3.3 无线网络速度慢（优化建议）

- **频段选择**：优先使用 **5 GHz**（Corp-WiFi），若设备仅支持 2.4 GHz，可申请临时切换到 `Corp-WiFi-2.4G`，但需注意该频段干扰较多。
- **避免后台大流量**：关闭非必需的云同步（如 OneDrive、百度网盘、迅雷）。大型文件传输建议使用有线网络。
- **检查信道干扰**：在 Windows 上用 `netsh wlan show networks mode=bssid` 查看信号强度；若附近有多个相同信道，可尝试让 IT 调整 AP 信道（需提交工单）。
- **设备兼容性**：部分老旧笔记本（2015 年以前）可能不支持 WPA2-Enterprise 的 EAP-PEAP，需更新网卡驱动或申请更换设备。

## 4. 网络故障排查

### 4.1 无法上网（完整排查流程）

> 适用于内网有线 / 无线 / VPN 连接后无法访问互联网。

1. **确认是全局故障还是部分网站**：尝试访问 `baidu.com` 和 `公司内网门户`（`http://portal.company.com`）。若仅部分网站打不开，可能是 DNS 问题（见步骤 5）。
2. **检查物理连接**：
   - 有线：查看网口指示灯是否亮起（绿色 = 千兆，橙色 = 百兆）。重新插拔网线，更换已知良好的网口。
   - 无线：检查 WiFi 图标是否显示已连接。尝试断开重连。
3. **重启设备与网络设备**：
   - 重启电脑 → 重启楼层交换机（非技术人员可跳过） → 若使用 USB 网卡，拔出重插。
4. **释放并更新 DHCP**：
   - 打开管理员命令提示符（Win+R → cmd → 右键以管理员运行）。
   - 依次执行：
     ```cmd
     ipconfig /release
     ipconfig /renew
     ipconfig /flushdns
     ```
5. **更换 DNS**：
   - 打开网络适配器属性 → IPv4 → 高级 → DNS 设置。
   - 首选 DNS 改为 `223.5.5.5`（阿里公共 DNS），备用改为 `119.29.29.29`（腾讯公共 DNS）。
6. **检查代理设置**：
   - 确保未启用系统代理（设置 → 网络 → 代理 → 关闭“使用代理服务器”）。
   - 检查浏览器插件（如 Proxy SwitchyOmega）是否错误配置了代理。
7. **联系 IT 报修**：如果以上步骤仍无法解决，通过 IT 工单系统提交“无法上网”工单，附带以下信息：
   - 计算机名、IP 地址（`ipconfig /all` 结果截图）
   - 发生时间、具体错误提示
   - 已执行过的排查步骤

### 4.2 网络卡顿（常见原因与解决方案）

| 原因 | 特征 | 解决方案 |
|------|------|----------|
| **带宽滥用** | 某台设备持续高上传/下载（如迅雷、百度网盘同步） | 通过路由器管理后台（192.168.1.1）查看连接设备流量；或联系 IT 使用流量分析工具定位。**禁止**在公司网络使用 P2P 下载。 |
| **无线干扰** | 2.4G 频段尤其明显（信道重叠、微波炉干扰） | 切换到 5 GHz 网络；或申请关闭附近 AP 的 2.4G 射频（需无线规划团队处理）。 |
| **内部 ARP 攻击** | 突然出现大面积断连或慢速，且伴有 IP 冲突提示 | 立即断开网络并通知 IT 安全团队（热线 400-xxx-xxxx），由安全人员使用 Wireshark 分析。 |
| **设备性能不足** | 办公笔记本配置较老（如 4GB 内存），同时运行多个应用 | 建议关闭非必要应用；或申请升级设备至 16GB 内存。 |

### 4.3 新增：网络故障分级与响应时间

| 故障等级 | 定义 | 响应时间 | 解决时间 |
|----------|------|----------|----------|
| **P1（紧急）** | 公司网络大面积中断，影响超 50% 员工 | 15 分钟 | 2 小时 |
| **P2（严重）** | 单楼层/部门内网中断，或 VPN 无法连接 | 30 分钟 | 4 小时 |
| **P3（一般）** | 个别用户无法上网、网速慢、WiFi 认证失败 | 2 小时 | 8 小时（工作时间） |
| **P4（低优先级）** | 网络改造建议、新设备接入申请 | 24 小时 | 5 个工作日 |

## 5. 网络安全注意事项

以下行为**严格禁止**，违者将按《信息安全管理制度》处理：

- **禁止**使用外接 WiFi 路由器、随身 WiFi、USB 网卡共享给他人。
- **禁止**在公司网络内设置任何形式的网络代理（HTTP Proxy、SOCKS5）或第三方 VPN（如 ExpressVPN、Shadowsocks），除非获得信息安全委员会书面授权。
- **禁止**将个人设备通过 USB 绑定或无线热点方式接入内网（如手机 USB 共享网络给电脑）。
- **禁止**通过访客网络访问内网系统（包括 VPN 隧道），访客网络与内网有物理隔离。
- **发现**网络异常卡顿、ARP 攻击、未知设备接入，**立即**断开本机网络并拨打 IT 安全紧急热线（400-xxx-xxxx）。
- **员工离职**时，IT 将在 24 小时内禁用其网络账户（包括 WiFi、VPN、有线登入权限）。部门负责人需在离职流程中提交网络清理确认。

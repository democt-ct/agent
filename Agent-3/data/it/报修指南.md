# 报修指南

**版本：** v2.2 | **生效日期：** 2026年5月19日 | **归口部门：** 信息技术部  

---

## 1. 目的与适用范围

本指南旨在规范公司员工对 IT 相关设备与系统的报修流程，明确故障分级标准、响应时效及自排查要求，确保故障得到高效、有序的处理。适用范围为全体员工使用的标准办公设备、软件及网络服务。**涉及服务器、网络核心设备、机房基础设施的故障**，请直接拨打 IT 值班热线（见 2.2 节），不得通过普通工单提交。

---

## 2. 报修范围

以下设备/系统故障可通过本指南报修，**非 IT 管理范围内的设备（如个人手机、家用路由器）不在支持之列**：

| 类别 | 适用范围 | 举例 |
|------|---------|------|
| 电脑故障 | 台式机、笔记本、瘦客户机 | 蓝屏（错误代码 0x0000007E 等）、死机、系统缓慢（启动时间 >5 分钟）、无法开机、硬盘异常（异响、数据丢失前的频繁卡顿） |
| 网络故障 | 有线/无线网络、VPN、互联网访问 | 无法上网（所有网页打不开）、VPN 连接失败（错误 769、800 等）、打印机/IP 电话无法访问、网络延迟 >500ms |
| 软件问题 | 公司授权软件（如 Office 365、ERP、CRM、邮件客户端） | Office 崩溃（Word 白屏）、邮件无法收发（Outlook 停止响应）、ERP 登录异常（密码过期、界面卡死）、浏览器插件冲突 |
| 外设故障 | 显示器、键盘、鼠标、耳机、USB 集线器 | 显示器黑屏（无信号）、键盘失灵（多键无响应）、鼠标不识别（光标不动）、USB 端口供电不足 |
| 账号问题 | 企业账号、Azure AD、邮箱、VPN 账号 | 账号锁定（多次输错密码）、密码重置、权限申请（需要开通特定系统模块） |

> **特别说明**：如故障同时影响 **≥3位相邻使用者** 或 **整个部门**，请立即拨打值班热线，不要提交工单，以免延误处理。

## 3. 报修方式

### 3.1 线上报修（首选）

**适用场景**：非紧急故障（P3 及以上）。完整流程如下：

```
步骤：
① 访问 IT 服务台：https://it.company.com
② 使用企业账号登录（SSO 自动认证，若无法登录请电话报修）
③ 点击「提交工单」→ 选择故障类别（参照第2节表格）
④ 填写详细描述，必须包含以下信息：
    - 故障发生时间（精确到分钟）
    - 是否尝试过重启、重装、切换网络
    - 错误提示文字或代码（如有截图请上传）
    - 影响范围（仅本人/多人）
⑤ 上传截图/日志（建议用工具截取完整屏幕而非手机拍照）
⑥ 提交后系统发送工单号至企业微信，请保留工单号便于后续跟进
```

**注意事项**：  
- 提交工单后，系统自动根据故障类别分配工程师，**无需重复提交**。  
- 工作日 18:00 后提交的工单，计入下一个工作日的响应时效。

### 3.2 电话报修（紧急）

**适用场景**：P1/P2 级别故障（多人无法工作、服务器宕机、数据泄露风险、单人关键业务完全中断）。

- **IT 服务热线**：400-xxx-xxxx（工作日 8:30 ~ 18:00）  
- **值班手机**：138-xxxx-xxxx（非工作时间、周末、节假日 *仅处理* P1 紧急故障，需有部门负责人确认）  
- **通话必备信息**：姓名、工号、故障现象、已尝试的解决步骤、当前是否影响其他人。

### 3.3 现场报修

**适用场景**：不方便线上/电话提交，或需携带设备至 IT 部检测。

- 地点：总部大楼 B1 层 IT 前台（工作日 9:00 ~ 17:00）  
- 流程：前台领取《报修登记表》→ 填写故障描述及联系方式 → 领取排队号 → 等待工程师叫号。  
- **注意**：现场报修仅接受硬件故障（键盘、鼠标、显示器等），软件问题建议线上提交。

## 4. 故障分级与响应时效

故障等级根据 **影响范围** 和 **紧急程度** 划分，工程师将按 SLA 目标处理。**未在规定时间内解决时，自动升级至主管工程师**。

| 级别 | 定义 | 响应时效 | 解决目标 | 升级条件 |
|------|------|---------|---------|---------|
| **P1 紧急** | 关键业务全线中断（如 ERP 宕机、全公司网络瘫痪、数据泄露风险）；或影响 ≥10 人正常办公 | 30 分钟内 | 4 小时内恢复 | 2 小时未解决 → 升级至 IT 总监 |
| **P2 高** | 单人关键业务受阻（无法登录 ERP、邮件完全不能收）；或影响 3~9 人 | 1 小时内 | 8 小时内解决 | 4 小时未解决 → 升级至高级工程师 |
| **P3 中** | 外设故障、软件崩溃、非关键系统缓慢，不阻断工作 | 4 小时内 | 1~3 个工作日 | 2 个工作日未解决 → 升级至主管 |
| **P4 低** | 咨询类、非紧急需求（如如何安装软件、查询软件许可） | 1 个工作日内 | 3~5 个工作日 | 无升级机制，但超 5 个工作日可投诉 |

> **响应时效**：指工程师首次联系用户的时间（电话或工单回复）。**解决目标**：指故障恢复或提供临时方案（如备用设备）的时间。

## 5. 常见故障自排查

**重要原则**：提交报修前，请务必完成自排查步骤，否则工程师可能退回工单要求补充尝试。

### 5.1 电脑蓝屏（Windows）

```
自排查步骤：
① 记录蓝屏错误代码（例如 0x0000007E, 0x0000001A, 0x00000050），可拍照留存
② 记下蓝屏出现的时间点（是否在安装软件/更新后？是否在插拔外设后？）
③ 尝试安全模式启动：开机时按 F8 → 选择「安全模式」（Win10/11 可能需要强制关机两次后再进入高级修复菜单）
④ 如能进入安全模式，卸载最近安装的软件（尤其是驱动、杀毒软件、系统更新）
⑤ 如频繁蓝屏（每周≥3次），先备份重要数据至网络盘或移动硬盘，再联系 IT 报修
```

**补充说明**：常见蓝屏代码含义：  
- `0x0000007B`：硬盘驱动问题（可能是引导错误）  
- `0x000000D1`：网卡或显卡驱动冲突  
- `0x0000009F`：电源管理或驱动不兼容

### 5.2 无法连接 VPN

```
自排查步骤：
① 确认已连网（能打开任意外部网站，例如 www.baidu.com）
② 确认 VPN 客户端版本为最新：Help → Check Update，如有更新请升级
③ 重启 VPN 客户端：完全关闭进程（任务管理器结束），等待10秒后再启动
④ 删除旧连接重新添加：设置 → 连接管理 → 删除当前配置 → 重新输入服务器地址和凭据
⑤ 检查本地防火墙或杀毒软件：临时关闭所有安全软件（如 360、腾讯管家、公司强制安装的 McAfee），再次连接
⑥ 以上均无效，提交工单并注明错误提示文字（例如“错误 629：端口已断开”或“错误 741：数据加密失败”）
```

### 5.3 打印机无法使用

```
自排查步骤：
① 确认打印机电源灯常亮、网络指示灯闪烁（若灯不亮检查电源线或网络线）
② 在控制面板 → 设备和打印机，找到对应打印机
③ 右键 →「设置为默认打印机」（确保不是“脱机使用打印机”状态）
④ 如仍无法打印，右键 →「打印疑难解答」（Windows 内置诊断工具）
⑤ 检查打印机的 IP 是否能 ping 通：Win+R → cmd → ping 打印机IP地址（需向 IT 索取或查看打印机屏幕）
⑥ 检查打印机驱动是否正常：设备管理器中是否有黄色感叹号，如有则重装驱动
⑦ 以上无效，联系 IT 报修，并提供打印机 IP 及故障现象描述
```

## 6. 常见问题 FAQ

| 问题 | 回答 |
|------|------|
| **报修后多久会有工程师联系我？** | 根据故障等级，P1 在 30 分钟内，P2 在 1 小时内，P3 在 4 小时内，P4 在下一个工作日结束前。请保持企业微信和手机畅通。 |
| **我可以指定工程师吗？** | 系统根据故障类型自动分配，无法指定。如对工程师服务不满，可通过投诉渠道反馈（见第8节）。 |
| **报修后如何查看进度？** | 可通过 IT 服务台网站「我的工单」查看状态（待分配、处理中、已关闭），或联系企业微信机器人输入工单号。 |
| **自排查步骤全部尝试仍无效怎么办？** | 请提交工单时注明“已完成所有自排查步骤”，工程师会优先处理，不会因为未尝试而退回。 |
| **我的电脑需要重装系统，是否在报修范围？** | 是，但请先备份个人数据至网络盘或移动硬盘。IT 不负责恢复非公司标准软件的个人文件。 |
| **非工作时间故障如何报修？** | 除 P1 紧急故障外，其他故障请通过线上提交工单，次日处理。P1 紧急请拨打值班手机。 |

## 7. 报修注意事项

- **信息完整**：提交工单时，务必按模板完整填写，包括故障发生时间、错误提示、已尝试步骤、受影响人数。信息不完整将导致退回补充。  
- **禁止操作**：任何涉及 **硬件拆卸（更换内存、硬盘、电池）** 的行为均需 IT 工程师现场操作。员工自行拆卸将 **失去该设备的保修资格**，并可能承担维修费用。  
- **数据备份**：在报修前，请自行将重要工作文件备份至公司网络盘（如 OneDrive/共享服务器）。IT 工程师维修过程中若需重装系统，不负责恢复未备份的个人数据。  
- **紧急处理**：如故障影响多人（≥3人），请务必直接拨打热线电话，不要等待工单响应，以免延误恢复时间。  
- **设备借用**：对于 P1/P2 故障且无替代方案时，IT 可提供备用设备（笔记本、键盘、鼠标等），但需在 3 个工作日内归还并完成数据迁移。

## 8. 投诉与建议

若对 IT 服务的响应速度、解决质量或工程师态度不满意，可通过以下渠道反馈：

- **邮件**：it-feedback@company.com（工作日 24 小时内回复）  
- **内网投诉区**：企业微信 → 工作台 → IT 服务 → 服务评价（匿名可选）  

建议内容请注明工单号或故障时间以便快速核查。**所有投诉将在一个工作日内转交 IT 总监并反馈处理结果。**

# 软件安装申请与管理规范

**生效日期：** 2026-05-19 | **归口部门：** 信息技术部 | **适用范围：** 全体员工及临时外聘人员

---

## 1. 目的与适用范围

### 1.1 目的
规范公司计算机设备（含台式机、笔记本、虚拟机及公司配发移动设备）上软件的安装行为，确保：
- **安全合规**：防范恶意软件、盗版软件及数据泄露风险
- **资产管控**：准确掌握软件资产清单，减少无授权使用
- **效率保障**：避免非必要软件占用系统资源或带宽

### 1.2 适用范围
- 所有由公司统一采购、配发或归公司管理的终端设备
- 员工自带的个人设备若接入公司网络或处理公司数据，同样需遵守本制度（以《BYOD安全管理细则》为准）

### 1.3 定义与关键条件

| 术语 | 定义 |
|------|------|
| **许可软件** | 已由公司统一采购并获得合法授权的商业软件，包含批量授权（VL）、订阅授权（SaaS）或永久许可。 |
| **白名单软件** | 经信息技术部安全评估并列入《企业软件许可目录》后，允许员工按流程安装的软件。 |
| **黑名单软件** | 因违反安全、法律或效率原则而禁止安装的软件（详见第4节）。 |
| **紧急安装** | 因工作急需（如客户演示、漏洞修复），需在指定时限内完成的安装请求，特指已授权但尚未部署的软件。 |

> 安装任何软件前，必须先确认该软件是否在 **《企业软件许可目录》** 内；不在目录中的软件需经过 **新软件评估申请**（流程见第2.3节）。

## 3. 软件安装申请流程

### 3.1 标准申请流程（3个工作日内完成）

适用于所有需要上级审批或安全审核的软件（包括2.2节和2.3节软件）。

```
步骤①：员工在 OA系统 → IT服务 → 新建申请 → 软件安装申请
步骤②：填写申请表单（必填项）：
  - 姓名、工号、部门、岗位
  - 所需软件名称、精确版本号（如Visual Studio Code 1.95）
  - 用途说明（至少50字，说明用于哪个项目或任务）
  - 期望安装日期（不得早于提交日期+2个工作日）
步骤③：系统自动发送审批通知给直接上级（或指定审批人）
  - 审批人需在1个工作日内完成审批
  - 拒绝时需填写理由（如：该工作可用替代软件完成）
步骤④：IT安全审核（仅涉密或高风险软件）
  - 审核内容：软件来源、版本漏洞、权限要求、隐私合规
  - 审核周期：1个工作日
  - 如发现高危漏洞，要求使用替代版本或沙箱安装
步骤⑤：审核通过后，IT工程师根据软件类型选择安装方式：
  - 标准软件（已封装）：通过MDM/SCCM远程静默安装，1个工作日内完成
  - 复杂软件（需交互配置）：邮件通知员工至IT部现场安装（需携带工牌）
步骤⑥：安装完成后，员工在OA上点击“确认接收”，系统自动登记资产并更新许可使用量。
```

#### 3.1.1 标准申请示例（开发类）

> **场景：** 后端工程师张工需要安装 PyCharm Professional 用于“库存管理系统”的开发。  
> **操作：**  
> 1. 张工在OA提交申请，选择软件“PyCharm Professional”，版本“2024.1”，用途“开发库存管理系统接口，需要使用Django框架，PyCharm提供智能提示和调试功能”。  
> 2. 上级赵经理审批通过（1小时后）。  
> 3. 因PyCharm不在安全审核清单中，跳过第4步。  
> 4. IT工程师通过远程推送安装包，并绑定张工的JetBrains企业账号（已有15个许可剩余）。  
> 5. 张工测试后点击“确认接收”。

### 3.2 紧急安装流程（适用于已采购但未安装的软件，半工作日内处理）

紧急安装仅适用于以下条件：
- **软件已在《企业软件许可目录》内**
- **员工因客户紧急需求、安全补丁或项目关键里程碑，无法等待标准流程**
- 需在OA申请时选择“紧急”并上传理由证据（如客户邮件截图、安全漏洞公告）

```
操作流程：
① 员工发送邮件至 it-software@company.com，标题格式：【紧急安装】姓名+工号+软件名
   正文附上OA紧急申请编号（需先提交OA申请，否则视为无效）
② IT热线（400-xxx-xxxx）同步接收工单，
   值班工程师在30分钟内响应
③ IT工程师在4小时内完成安装（远程或现场）
④ 后续补签《紧急安装确认单》，并记录在年度审计中
```

> **注意：** 紧急流程每月每人限用2次，超出需部门总监特批。

### 2.3 新软件评估申请（目录外软件）

若所需软件不在《企业软件许可目录》内，员工需先发起 **新软件评估**：

1. 在 OA → IT服务 → 新建申请 → 新软件评估
2. 提交软件名称、版本、官网链接、功能说明、所需许可数量
3. 信息技术部在 **5个工作日** 内完成安全评估、许可评估、替代性分析
4. 评估结论：
   - **批准采购**：提交采购流程，采购完成后纳入目录
   - **拒绝**：给出理由（如功能重叠、安全风险）
   - **建议替代**：推荐目录内已有类似软件

## 4. 软件白名单与黑名单

### 4.1 白名单（自动允许安装，无需审批）

以下类别软件默认允许安装，但仍需从官方源获取：
- 公司采购的所有许可软件（依据《目录》）
- 开源软件（需经安全审核，审核列表在IT知识库中发布，如Visual Studio Code、Git、7-Zip、Notepad++、VLC等）
- IT部门发布的内部工具（如VPN客户端、打印驱动、资产扫描工具）

> **例外：** 如开源软件被曝出重大漏洞（CVSS评分≥7.0），暂停白名单资格，直到补丁发布。

### 4.2 黑名单（严格禁止安装）

| 软件类别 | 典型软件示例 | 禁止原因 | 发现后的处理 |
|---------|-------------|---------|------------|
| 下载加速类 | 迅雷、网际快车、QQ旋风 | 抢占带宽（P2P上传）、常捆绑恶意插件 | 立即卸载，书面警告一次 |
| 远程控制类 | TeamViewer、AnyDesk、向日葵 | 绕过公司远程网关，存在数据泄露风险 | 立即卸载，书面警告并通报部门 |
| 盗版/破解软件 | 各类“绿色版”“注册机” | 违反《著作权法》，可能植入后门 | 立即卸载，书面警告，记过一次 |
| 游戏客户端 | Steam、LOL、王者荣耀PC版 | 影响工作效率，拖慢网络 | 立即卸载，口头警告；重复发现记过 |
| 代理/VPN（非公司授权） | Shadowsocks、Clash、VPN Gate | 绕过公司安全策略，访问不受控网络 | 立即卸载，严重警告，可能纪律处分 |
| 加密挖矿/区块链 | 以太坊矿工、Chia客户端 | 消耗大量计算资源，公司电费超标 | 立即卸载，永久记录，上报HR |
| 文件共享P2P | BitTorrent、eMule、uTorrent | 版权风险，病毒传播 | 立即卸载，书面警告 |

> **补充说明：** 黑名单不定期更新，IT部会在OA首页公告。员工若因工作需要必须使用此类软件（如TeamViewer用于远程支持客户设备），需提交 **《例外申请》** ，由IT安全总监和法务共同审批。

## 5. 违规处理与惩罚措施

### 5.1 违规分级

| 违规类型 | 处罚标准 | 备注 |
|---------|---------|------|
| **轻微违规** | 未经申请安装白名单外但非黑名单的免费软件（如WinRAR、XnView） | 口头警告，限期1个工作日卸载或补交申请 |
| **一般违规** | 安装黑名单软件（首次） | 书面警告，立即卸载，通报部门经理 |
| **严重违规** | 累计3次一般违规 / 安装盗版软件 / 使用公司设备挖矿 | 记过一次，绩效扣减当月10%，全公司通报 |
| **特别严重** | 因违规安装导致公司网络中断、数据泄露或法律诉讼 | 移交纪律委员会，可能解除劳动合同并追究法律责任 |

### 5.2 监督与检查

- IT部门每月进行一次 **终端软件扫描**，比对《企业软件许可目录》和黑名单
- 扫描结果通过OA报告至部门总监和HR
- 员工可匿名举报违规安装，举报查实给予奖励（如50元京东卡）

## 6. 常见问题 FAQ

### Q1：我是否可以安装绿色免安装版软件（如放U盘里的工具）？
**A：** 不可以。所有软件安装必须通过IT统一渠道（官方安装程序、企业应用中心或IT远程部署）。免安装版无法控制版本、无法确保无后门，属于“未经申请安装违规”。

### Q2：我申请了PyCharm但一个月后离职了，许可怎么处理？
**A：** IT部门会在离职流程中自动回收许可。你应在离职前主动确认软件已卸载（可在OA提交“设备归还”清单）。若发现未卸载，将影响最后薪资结算。

### Q3：我想试用一个不在目录里的商业软件，需要走什么流程？
**A：** 请先提交“新软件评估申请”（见2.3节）。评估通过后，IT会提供试用许可（通常30天），并设置到期自动禁用。不允许自行从官网下载试用版。

### Q4：我的电脑是Mac，申请流程和Windows一样吗？
**A：** 基本相同。但Mac上的软件安装方式多为自助式（从企业应用中心下载DMG）。安全审核标准一致。DMG必须经过IT签名或公证。

### Q5：如果我的紧急申请被拒绝，我可以直接让同事把他的安装包发给我吗？
**A：** 绝对禁止。任何软件都必须从官方源或IT内部源获取。同事转发安装包可能携带风险（如被篡改）。违反视为严重违规。

### Q6：软件安装后出现蓝屏/闪退，应该找谁？
**A：** 立即联系IT服务台（400-xxx-xxxx或OA提报IT工单）。不要自己卸载重装（可能破坏关联配置）。IT工程师会排查冲突并处理。

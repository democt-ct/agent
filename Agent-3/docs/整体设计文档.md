# 企业多专家 Agent 系统 — 整体设计文档

> 版本：v1.0 · 2026-05-27

---

## 一、产品定位

本系统是一个**对话驱动的企业智能工作台**，核心理念是：

- **知识问答类**：纯对话，Agent 直接回答
- **操作执行类**：对话 + 交互卡片，Agent 识别意图后渲染操作组件，用户一步完成

用户不需要学习菜单和表单，说话即可完成所有日常工作事务。

---

## 二、交互设计原则

### 2.1 两种交互模式

| 模式 | 触发条件 | 示例 |
|------|----------|------|
| **纯对话** | 知识查询、制度咨询、状态查询 | "年假政策是什么" "我的申请进度" |
| **卡片交互** | 有固定字段的操作类意图 | "我想请假" "报销一笔差旅费" "审批下属申请" |

### 2.2 卡片类型

| 卡片类型 | 触发意图 | 说明 |
|----------|----------|------|
| `leave_form` | 请假申请 | 类型/日期/原因，提交后直接写库 |
| `expense_form` | 报销申请 | 类型/金额/说明 |
| `approval_action` | 审批操作 | 通过/驳回按钮，附带申请摘要 |
| `attendance_punch` | 打卡 | 上班/下班一键按钮 |
| `overtime_form` | 加班申请 | 日期/时长/原因 |
| `trip_form` | 出差申请 | 目的地/日期/事由 |
| `quick_replies` | 多选确认 | Agent 给出选项，用户点选 |
| `confirm` | 二次确认 | 高风险操作前确认 |

### 2.3 技术实现

Agent 回复 JSON 中携带 `action_card` 字段，前端识别后在消息下方渲染对应组件：

```json
{
  "answer": "好的，请填写请假信息：",
  "action_card": {
    "type": "leave_form",
    "prefill": { "leave_type": "年假" }
  }
}
```

卡片提交后直接调用对应 REST API，结果以 Agent 消息形式展示在对话中。

## 四、审批流架构（通用化）

### 当前问题
审批规则硬编码在 `hr_tools.py`，每种业务单独写逻辑，扩展性差。

### 目标设计

```
approval_templates 表
├── id
├── name          # "请假审批-3天以内"
├── record_type   # leave / expense / overtime / trip
├── condition     # JSON: {"field": "total_days", "op": "<=", "value": 3}
└── steps         # JSON: [{"role": "direct_manager"}, {"role": "hr"}]
```

提交任何申请时，系统根据 `record_type` + 条件匹配模板，自动生成审批流。管理员可在后台可视化配置，无需改代码。

## 五、Agent 能力增强

### 5.1 意图识别增强

在 Orchestrator 路由层加操作意图识别：

| 用户说 | 识别意图 | 触发 |
|--------|----------|------|
| "我想请假" / "申请年假" | `intent=leave_apply` | 返回 `leave_form` 卡片 |
| "帮我报销" / "提交报销" | `intent=expense_apply` | 返回 `expense_form` 卡片 |
| "我要打卡" / "上班打卡" | `intent=punch_in` | 返回 `attendance_punch` 卡片 |
| "审批一下" / "我有待审批吗" | `intent=approval_check` | 返回待审批列表 + `approval_action` 卡片 |
| "年假政策" / "报销标准" | `intent=query` | 纯对话回答 |

### 5.2 上下文记忆

- 记住本次对话中产生的申请 ID，支持"通过刚才那个"
- 记住用户偏好（常用请假类型、常用报销类型）
- 跨会话记忆用户历史（"上次你请了 3 天年假"）

### 5.3 主动能力

- 登录时检查待办，主动汇报
- 假期余额低于阈值时主动提醒
- 审批超时时主动催促

## 八、设计约束

1. **对话优先**：所有操作都应该能通过对话完成，表单/按钮是辅助加速手段
2. **最小权限**：员工只能操作自己的数据，经理只能看下属，HR 全局可见
3. **审批不可绕过**：所有涉及资源的申请必须经过审批流，不允许直接修改状态
4. **操作可追溯**：所有工具调用写入 audit_log，支持事后审计

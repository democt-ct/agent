# 通用审批引擎

> 版本 v1.0 · 2026-06-10  
> 对应实现: `src/skills/approval_engine.py`

---

## 1. 概述

审批引擎统一了请假、报销（及未来更多类型）的审批流程，消除 `hr_tools.py` 和
`finance_tools.py` 中重复的审批逻辑。

核心思路：**规则配置 + 汇报链 = 审批步骤**。

## 2. 审批层级规则

### 2.1 请假审批

| 天数 | 审批层级 | 审批人 |
|------|---------|--------|
| ≤ 3 天 | 1 级 | 直属上级 |
| ≤ 7 天 | 2 级 | 直属上级 → 部门负责人 |
| > 7 天 | 3 级 | 直属上级 → 部门负责人 → HR 总监 |

### 2.2 报销审批

| 金额 | 审批层级 | 审批人 |
|------|---------|--------|
| < 5,000 元 | 1 级 | 直属上级 |
| ≥ 5,000 元 | 2 级 | 直属上级 → 财务经理 |

### 2.3 扩展新类型

在 `_get_approval_rules()` 添加新规则即可：

```python
DEVICE_APPROVAL_RULES = [
    (5000,  "direct_manager", "直属上级"),
    (float("inf"), "dept_head", "部门负责人"),
]
```

## 3. 审批流程

```
用户提交申请
      │
      ▼
submit_for_approval()
      │
      ├─ build_approval_chain() — 按规则 + 汇报链确定步骤
      ├─ db_create_approval_flow() — 写入 approval_flow 表
      ├─ 通知第一位审批人
      └─ 通知申请人
            │
            ▼
      审批人看到待审批
            │
      ┌─────┴─────┐
      ▼           ▼
  approve_step  reject_step
      │           │
      │           └─ 驳回整条记录 → 通知申请人
      │
      ├─ 检查是否所有步骤已通过
      │
      ├─ 否 → 通知下一位审批人
      └─ 是 → finalize:
              ├─ 更新主记录状态 (approved)
              ├─ 扣减假期余额 (请假)
              └─ 通知申请人
```

## 4. API 参考

### 4.1 `build_approval_chain()`

根据审批规则和汇报链构建审批步骤列表。

```python
build_approval_chain(
    record_type: str,          # "leave" | "expense"
    user_id: str,              # 申请人ID
    threshold_value: float,    # 请假天数/报销金额
    chain: list[dict] | None,  # 汇报链（可选，不传自动查）
) -> list[dict]
```

**返回**：
```json
[
  {"step": 1, "approver_id": "EMP002", "approver_role": "direct_manager"},
  {"step": 2, "approver_id": "EMP003", "approver_role": "hr"}
]
```

### 4.2 `submit_for_approval()`

提交审批 — 自动构建审批链 + 创建审批步骤 + 发通知。

```python
submit_for_approval(
    record_type: str,
    record_id: str,            # 关联记录ID
    user_id: str,
    threshold_value: float,
    extra: dict | None,        # {leave_type, expense_type, ...}
) -> dict
```

### 4.3 `approve_step()`

审批通过某个步骤，自动推进。

```python
approve_step(
    step_id: str,              # approval_flow.id
    approver_id: str,          # 审批人ID（身份校验）
    comment: str = "",         # 审批意见
) -> dict
```

**返回**：
```json
{
  "success": true,
  "step_id": "abc123",
  "step_number": 1,
  "all_approved": false,
  "message": "审批步骤 1/3 已通过，等待下一级审批"
}
```

### 4.4 `reject_step()`

驳回整条记录。

```python
reject_step(
    step_id: str,
    approver_id: str,
    comment: str = "",         # 驳回理由
) -> dict
```

### 4.5 `get_approval_status()`

查看某条记录的审批进度。

```python
get_approval_status(record_type: str, record_id: str) -> dict
```

**返回**：
```json
{
  "record_type": "leave",
  "record_id": "REC001",
  "total_steps": 2,
  "completed": 1,
  "steps": [
    {"step": 1, "approver": "李四", "role": "direct_manager", "status": "approved"},
    {"step": 2, "approver": "王五", "role": "hr", "status": "pending"}
  ]
}
```

### 4.6 `get_pending_approvals()`

某人（审批人）的所有待审批项，请假和报销合并。

```python
get_pending_approvals(approver_id: str) -> dict
```

## 5. 与现有工具的关系

| 现有工具 | 迁移方向 |
|---------|---------|
| `hr_tools.submit_leave_request` | 内部审批流部分可改为调 `approval_engine.submit_for_approval` |
| `hr_tools.approve_leave` | 可改为调 `approval_engine.approve_step` |
| `hr_tools.reject_leave` | 可改为调 `approval_engine.reject_step` |
| `hr_tools.get_approval_progress` | 可改为调 `approval_engine.get_approval_status` |
| `hr_tools.get_pending_approvals` | 可改为调 `approval_engine.get_pending_approvals` |
| `hr_tools.check_policy` | **独立保留**（合规检查不是审批流的一部分） |

现有工具 **保持可用**，可选择在后续迭代中内部转发到 `approval_engine`。

## 6. Agent 工具映射

| workflow 函数 | Agent 工具名 | 工具文件 |
|--------------|-------------|---------|
| `get_pending_approvals` | `get_pending_approvals` | `general_tools.py` |

其他审批操作（approve/reject）目前通过 `hr_tools.approve_leave` / `hr_tools.reject_leave` 暴露给 Agent，
后续可统一到通用审批工具。

## 7. CLI 使用

```bash
# 查看审批人的待审批列表
python scripts/approval_ops.py pending EMP002

# 审批通过
python scripts/approval_ops.py approve <step_id> EMP002 --comment "同意"

# 驳回
python scripts/approval_ops.py reject <step_id> EMP002 --comment "材料不全"

# 查看审批进度
python scripts/approval_ops.py status leave <record_id>
```

# 本地启动与内网穿透

本文档记录企业多专家 Agent 系统的本地 Web 启动方式，以及演示时可选的内网穿透方式。

## 1. 本地启动 Web 服务

本项目需要使用项目内的 `.venv` 虚拟环境运行。推荐直接调用 `.venv` 里的 Python：

```bash
.venv/Scripts/python.exe -m src.api.app
```

也可以先激活虚拟环境后再启动：

```bash
source .venv/Scripts/activate
python -m src.api.app
```

启动成功后访问：

```text
http://localhost:8080
http://localhost:8080/admin.html
```

如果 Poetry 环境可用，也可以使用：

```bash
poetry run python -m src.api.app
```

或使用 uvicorn：

```bash
.venv/Scripts/python.exe -m uvicorn src.api.app:app --host 0.0.0.0 --port 8080
```

## 2. 内网穿透

先确保本地服务已经在 `http://localhost:8080` 正常运行，然后另开一个终端启动穿透工具。任选一种已安装的工具即可。

### 方案 A：cloudflared 临时隧道

```bash
cloudflared tunnel --url http://localhost:8080
```

终端会输出一个公网访问地址，通常形如：

```text
https://xxxx.trycloudflare.com
```

把这个地址发给演示对象即可访问本地系统。

### 方案 B：ngrok

```bash
ngrok http 8080
```

终端会输出 `Forwarding` 地址，复制 `https://...ngrok-free.app` 访问即可。

### 方案 C：cpolar

```bash
cpolar http 8080
```

复制 cpolar 输出的公网地址访问。

## 3. 验证步骤

1. 启动本地服务：

   ```bash
   .venv/Scripts/python.exe -m src.api.app
   ```

2. 浏览器打开：

   ```text
   http://localhost:8080
   ```

3. 确认页面可以正常加载。

4. 另开终端启动内网穿透，例如：

   ```bash
   cloudflared tunnel --url http://localhost:8080
   ```

5. 复制终端输出的公网 `https://...` 地址，在浏览器中打开验证。

## 4. 注意事项

- 内网穿透会把本地服务暴露到公网。
- 演示结束后请关闭穿透进程。
- 不要在公网演示环境中使用真实敏感数据或生产密钥。
- 如果公网地址能打开页面但聊天接口失败，优先检查本地服务是否仍在运行，以及浏览器控制台/终端日志中的错误。

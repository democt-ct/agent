## 1. 系统概览

### 1.1 一句话描述

基于自研 ReAct 引擎 + Orchestrator 调度的**多专家 RAG Agent 系统**，3 个领域专家（HR / IT / 法务）各自拥有独立知识库和工具集，支持跨域串行编排。

### 2.2 ReAct 循环细节

```
AgentRequest 进入 BaseAgent.run()
    │
    ├─ ① kb.query(query, top_k=10)
    │      → 查询改写 (LLM): "年假什么时候发" → "年假发放规则 发放时间"
    │      → 向量检索 (Chroma):  语义相似度 top_k
    │      → BM25 检索 (jieba):  关键词匹配 top_k
    │      → RRF 融合 (k=60):    合并排序
    │      → [Reranker]:         精选 top_k=3 (如果启用)
    │      → 结果注入 System Prompt
    │
    ├─ ② _build_system_prompt(query, retrieved, handoff_context)
    │      → 角色定义: "你是HR专家..."
    │      → [用户画像]: department=技术部, role=工程师  (Memory)
    │      → [交割上下文]: 来自前序Agent的已处理结果
    │      → 知识库参考材料: 检索到的文档片段
    │      → 可用工具列表: get_leave_balance, submit_leave_request...
    │
    ├─ ③ ReAct 循环 (for _ in range(max_tool_calls))
    │      LLM 调用 (_call_llm_with_retry, 最多重试3次)
    │      ├─ msg.content && !msg.tool_calls → 返回最终回答 (status=success)
    │      └─ msg.tool_calls → 逐个执行工具
    │           ├─ _execute_tool(name, args)
    │           ├─ tool_call_log 记录
    │           ├─ [on_step("tool_call", {...})]  → SSE推送 + Trace记录
    │           └─ 结果回传 messages (含 reasoning_content)
    │
    └─ ④ 达上限 → 强制生成
           → messages.append("请基于以上结果给出最终回答")
           → LLM 调用 → 返回 (status=partial)
```

## 4. 核心功能详解

### 4.1 Orchestrator 三级路由

**确定性优先：规则 → 关键词 → LLM**

| 层级 | 触发条件 | confidence | 延迟 |
|------|---------|------------|------|
| 寒暄检测 | 正则匹配 "你好/谢谢/再见/你是谁" | 1.0 (固定) | <1ms |
| 关键词单领域 | 仅一个领域命中关键词 | 0.76~0.95 (动态) | <1ms |
| 关键词多领域(差距≥2x) | 主领域命中数 ≥ 次领域×2 | 0.70~0.85 (动态) | <1ms |
| 关键词多领域(位置优先) | 多领域命中数相近 | 0.60~0.80 (动态) | <1ms |
| LLM 路由 | 以上均未命中 | LLM 返回 (动态) | ~500ms |

**关键词库**：3 领域共 87 个关键词，大小写不敏感匹配。

**动态 confidence 公式**：

```
单领域:  0.75 + 0.20 × (命中数 / 领域总词数)    → 上限 0.95
差距≥2x: 0.70 + 0.10 × min(倍数-1, 1.5)       → 上限 0.85
位置优先: 0.60 + 0.20 × (1 - 首词位置/查询长度)  → 上限 0.80
```

### 4.2 RAG 检索链路

```
KnowledgeBase.query(query, top_k=10)
  │
  ├─ 缓存检查 (TTL 60s, MD5 key)
  │
  ├─ 查询改写 (LLM, 可选)
  │     "年终奖什么时候发" → "年终奖发放规则 发放时间"
  │
  ├─ Embedding: BAAI/bge-large-zh-v1.5 (1024d)
  │
  ├─ 向量检索: Chroma collection.query() → top_k (或 rerank_retrieval_k=20)
  │
  ├─ BM25 检索: rank_bm25 + jieba 分词 → top_k (或 rerank_retrieval_k=20)
  │
  ├─ RRF 融合 (k=60, BM25 权重 1.5)
  │     score = 1/(60+rank_vector+1) + 1.5/(60+rank_bm25+1)
  │
  ├─ [重排序] (RERANK_ENABLED=true)
  │     CrossEncoder(query, chunk) → 重新打分 → top_k=3
  │
  └─ 返回 [{"content", "source", "score", "method"}, ...]
```

### 4.3 工具系统

**ToolDef 格式**：

```python
ToolDef(
    name="get_leave_balance",
    description="查询员工的年假、病假和事假剩余天数。",
    parameters={  # JSON Schema
        "type": "object",
        "properties": {"user_id": {"type": "string"}},
        "required": ["user_id"],
    },
    implementation=get_leave_balance,  # Python 函数
)
```

**Agent 工具调用流程**：

```
LLM 返回 tool_calls[{function: {name, arguments: "{}"}}]
  → json.loads(arguments)
  → tool_map[name].implementation(**args)
  → 结果回传 messages (role=tool)
  → LLM 继续推理
```

**可用工具清单**：

| Agent | 工具 | 参数 | Mock 行为 |
|-------|------|------|-----------|
| HR | `get_leave_balance` | user_id | 查内存字典 (3 条员工数据) |
| HR | `submit_leave_request` | user_id, leave_type, start_date, end_date, reason | 追加到内存列表 |
| IT | `check_ticket_status` | ticket_id | 查内存字典 (2 条工单) |
| IT | `create_ticket` | user_id, issue_type, description, priority | 创建新工单 (TK003+) |
| IT | `check_device_inventory` | device_type | 查内存字典 (5 种设备) |
| 法务 | `search_contract` | keyword | 关键词匹配 |
| 法务 | `check_compliance` | doc_summary | 合规性模拟评估 |

### 4.4 跨域交割 (Handoff)

**流程**：当路由识别 `secondary` Agent 或 Agent 返回 `handoff_suggestions` 时触发。

**交割上下文格式**：

```json
{
  "from_agent": "hr_agent",
  "summary": "已处理病假审批，按80%工资计算",
  "context": {"query": "...", "already_answered": ["病假已审批"]},
  "status": "success",
  "handoff_type": "chain"
}
```

**汇总格式**：两个 Agent 的回答用 `---` 分隔。

### 5.3 Reranker — 检索重排序

### 7.1 端点总览

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/chat` | 同步对话，返回完整 JSON |
| `GET` | `/api/chat/stream?query=...&session_id=...` | SSE 流式对话 |
| `GET` | `/api/agents` | 获取所有 Agent 信息 |
| `GET` | `/api/health` | 健康检查 |
| `GET` | `/` | 前端页面 (StaticFiles) |

### 7.2 POST /api/chat

**请求**：

```json
{
  "query": "年假还剩几天",
  "session_id": "abc123",
  "history": [
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！我是企业智能助手..."}
  ]
}
```

**响应**：

```json
{
  "session_id": "abc123",
  "answer": "根据查询结果，您目前年假剩余 7 天...",
  "routing": {
    "primary": "hr_agent",
    "secondary": null,
    "confidence": 0.76,
    "method": "keyword",
    "matched_keywords": ["年假"]
  },
  "agent_name": "hr_agent",
  "retrieved_chunks": [{"source": "年假管理制度.md", "score": 0.94, "content": "..."}],
  "tool_calls": [{"name": "get_leave_balance", "arguments": {"user_id": "EMP001"}, "result": {...}}],

  "processing_time_ms": 1250,

}
```

### 7.3 SSE /api/chat/stream

**事件序列**：

```
event: routing     → {"primary": "hr_agent", "confidence": 0.76, ...}
event: retrieval   → {"count": 8, "sources": [{"source": "...", "score": 0.94}]}
event: tool_call   → {"name": "get_leave_balance", "arguments": {...}, "result": {...}}
event: reasoning   → {"content": "用户想查询年假余额..."}
event: answer      → {"content": "根据查询结果...", "agent_name": "hr_agent", "tokens": 850}
event: done        → {"session_id": "abc123"}
```

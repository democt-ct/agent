# 企业级 Multi-Agent 智能协作平台（面试项目说明）

## 项目背景

传统企业 AI 助手大多停留在单轮问答阶段，缺乏任务拆解、工具调用、多系统协同以及长期记忆能力，难以真正参与企业工作流。为了解决这一问题，我设计并实现了一个面向企业场景的 Multi-Agent 智能协作平台，通过“业务领域 Agent + 通用能力中台”的架构，实现复杂任务的自动规划与执行。

---

# 项目目标

构建一个具备：

- 多 Agent 协同
- 长上下文记忆
- 企业知识库检索（RAG）
- 工具调用
- 工作流编排
- 多系统接入

能力的企业级 AI 智能助手平台。

---

# 系统架构设计

## 一、业务领域 Agent（Domain Agents）

根据企业真实组织结构设计不同业务 Agent：

### HR Agent

负责：

- 招聘问答
- 简历筛选
- 员工制度查询
- 考勤与请假流程

### 法务 Agent

负责：

- 合同审核
- 风险提示
- 法务知识问答
- 条款解析

### IT Agent

负责：

- 工单处理
- 权限申请
- 系统故障排查
- 运维知识支持

### 财务 Agent（可扩展）

负责：

- 报销流程
- 财务制度问答
- 数据统计分析

---

## 二、通用能力中台（Core Infrastructure Services）

为避免各业务 Agent 重复实现底层能力，系统抽象出一组轻量级基础设施模块。这些模块**不继承 BaseAgent、不做 ReAct 推理**，而是作为确定性服务被业务 Agent 按需调用，兼顾效率与可控性。

### Planner 规划模块

负责：

- 用户意图分析
- 多步骤任务拆解
- DAG 工作流规划
- Agent 调度

例如：
“帮我分析候选人并生成面试报告”
会被拆解为：

1. 简历解析
2. 岗位匹配
3. 技能分析
4. 生成评价报告

实现方式：关键词 fast-path + LLM JSON 输出 → 拓扑排序，不参与 ReAct 循环。

---

### Retrieval 检索模块

负责：

- 企业知识库检索
- RAG 召回
- 文档向量搜索
- 上下文增强

支持：

- PDF
- Word
- 企业内部文档
- FAQ

实现方式：向量 + BM25 混合检索 (RRF) → Cohere Rerank 重排序，嵌入在 BaseAgent._react_loop() 的第一步。

---

### Tool Gateway 工具网关

负责：

- API 调用
- 数据库操作
- 权限校验与审计
- 外部系统通信

可接入：

- OA
- CRM
- 邮件系统
- 飞书/钉钉
- GitLab
- 数据分析工具

实现方式：单一 execute() 入口，先校验 SessionContext 权限，再执行 implementation，审计日志写入 audit_log 表。

---

### Memory 记忆模块

负责：

- L2 会话存储 (SQLite conversation_store)
- L3 长期记忆管理 (long_term_memory 表)
- 用户画像维护
- LLM 摘要压缩

实现方式：会话超阈值时触发 SummaryCompressor 生成摘要替换旧消息；工作流完成后启发式提取关键事实写入长期记忆。

---

### Review 审查模块

负责：

- 结果校验（规则层，不调 LLM）
- 数值一致性检查
- 制度溯源验证
- 编造/幻觉检测

实现方式：正则匹配回答中的数值与工具返回比对 + 条款引用与检索来源比对，低置信回答自动标注”仅供参考”。

# 核心亮点

## 1. 多 Agent 协同架构

相比传统单 Agent 系统，本项目实现了：

- Agent 间任务分发
- 工作流协同
- 状态传递
- 多角色协作

---

## 2. 企业级工作流能力

系统不仅支持聊天，还支持：

- 多步骤任务执行
- 工具链调用
- 自动任务规划
- 复杂流程编排

---

## 3. 长上下文与记忆机制

通过 Memory 记忆模块实现：

- L2 会话存储 + L3 长期偏好记录
- LLM 摘要压缩（超阈值自动触发）
- 历史任务状态管理

---

## 4. RAG + Tool Calling 深度结合

实现：

- 企业知识库增强
- API 实时数据获取
- 外部系统联动

降低模型幻觉问题。

---

## 5. 可扩展架构设计

采用：

> “领域 Agent + 能力中台（轻量服务）”

架构后，可以快速新增：

- 医疗 Agent
- 销售 Agent
- 客服 Agent
- 数据分析 Agent

新增 Agent 只需继承 BaseAgent、绑定知识库和工具即可，无需重复开发检索/记忆/审查等底层能力。

# 项目难点

## 1. Agent 协同调度

解决多个 Agent 间：

- 通信
- 状态同步
- 上下文共享

问题。

---

## 2. 长上下文管理

通过：

- 历史摘要
- 向量记忆
- 分层上下文

降低 Token 消耗。

---

## 3. 幻觉控制

结合：

- RAG 知识库约束
- Review 审查模块（规则层数值校验 + 制度溯源）
- 工具返回值交叉校验
- 路由规则兜底

提升输出可信度。

### Task 1 · 跨域合并回答综合摘要

**文件**：`src/api/service/chat_service.py`（第 259-265 行）

**现状**：两个 Agent 并行执行后直接字符串拼接，回答割裂，用户体验差。

**目标**：并行结果出来后，追加一次 LLM 调用做综合摘要，输出连贯的统一回答。

**改动步骤**：

1. 在 `chat_service.py` 新增 `_merge_answers()` 函数：

```python
def _merge_answers(client, model, query, results: list[dict]) -> str:
    parts = "\n\n".join(
        f"[{r['agent_name']}]\n{r['answer']}" for r in results
    )
    messages = [
        {"role": "system", "content": "你是一个信息整合助手，将多个专家的回答整合为一段连贯、无重复的回答。"},
        {"role": "user", "content": f"用户问题：{query}\n\n各专家回答：\n{parts}"},
    ]
    resp = call_llm_with_retry(client, model=model, messages=messages, temperature=0.3)
    return resp.choices[0].message.content or parts
```

2. 在并行执行分支（第 252-293 行）和 handoff 分支（第 315-367 行）的合并处，将 `merged_answer` 字符串拼接替换为调用 `_merge_answers()`。
3. 原始拼接结果保留在 `result["raw_answers"]` 供 trace 查看。

**验收**：问"出差报销需要提前请假吗"，返回一段整合了 HR 和财务视角的连贯回答，不出现 `---` 分隔线。

### Task 2 · 用户上下文移出 user 消息

**文件**：`src/api/service/chat_service.py`（第 82-88 行）、`src/agents/base_agent.py`（第 120-124 行）

**现状**：用户身份通过字符串拼接注入到 `user` 消息内容，污染对话历史和 token 计数。

**目标**：用户上下文作为独立 `system` 消息注入，不进入 `user` 消息。

**改动步骤**：

1. `AgentRequest` 新增字段 `user_context: str = ""`。
2. `chat_service.py` 中将 `user_hint` 存入 `req.user_context`，不再拼接到 `query`。
3. `base_agent.py` 的 `_react_loop()` 中，在 `messages.append({"role": "system", ...})` 之后，若 `request.user_context` 非空，追加第二条 system 消息：

```python
if request.user_context:
    messages.append({"role": "system", "content": request.user_context})
```

4. `conversation_store.save()` 存储时使用原始 `raw_query`，不含 user_hint。

**验收**：查看 trace，user 消息内容不含 `[当前登录用户...]` 字样；工具调用 user_id 仍正确。

### Task 5 · 通用快速查询路径

**文件**：`src/api/service/chat_service.py`（第 457-491 行）、`src/tools/base.py`

**现状**：`_try_direct_query_tool` 只为 HR 假期查询硬编码，其他 Agent 的高频简单查询都走完整 ReAct。

**目标**：在 `ToolDef` 上声明快速匹配规则，框架统一处理，不需要为每个 Agent 手写。

**改动步骤**：

1. `ToolDef` 新增字段：

```python
@dataclass
class ToolDef:
    ...
    quick_triggers: list[str] = field(default_factory=list)  # 触发关键词
    quick_args_builder: Callable | None = None               # 构造参数的函数
```

2. `chat_service.py` 中 `_try_direct_query_tool` 改为通用 `_try_quick_tool()`：

```python
def _try_quick_tool(agent_key, raw_query, session_ctx, agent_instances):
    agent = agent_instances.get(agent_key)
    for tool in (agent.tools if agent else []):
        if tool.quick_triggers and tool.quick_args_builder:
            if any(kw in raw_query for kw in tool.quick_triggers):
                args = tool.quick_args_builder(session_ctx)
                result = tool.implementation(**args)
                return {"answer": ..., "tool_calls": [...]}
    return None
```

3. HR 假期查询、IT 工单状态查询、财务报销查询等高频工具补充 `quick_triggers` 和 `quick_args_builder`。

**验收**："我的工单状态"、"我的报销申请"直接返回工具结果，不调用 LLM；添加新快速查询只需在 ToolDef 声明，无需改 chat_service。

### Task 6 · 激活 SummaryCompressor

**文件**：`src/api/service/chat_service.py`（第 431-443 行）、`src/memory/summary_compressor.py`

**现状**：`_try_compress()` 只打 log，SummaryCompressor 从未实际调用。长期会话 context 无限增长。

**目标**：会话超过 20 条消息时，异步压缩早期历史为摘要。

**改动步骤**：

1. `app.py` 初始化时构建 `SummaryCompressor(client=llm_client)` 并挂在 `state.compressor`。
2. `_try_compress()` 改为真实压缩调用：

```python
def _try_compress(store, session_id, compressor):
    if compressor is None: return
    count = store.count_messages(session_id)
    if count <= 20: return
    import threading
    threading.Thread(
        target=compressor.compress, args=(store, session_id), daemon=True
    ).start()
```

3. `chat_service.py` 中将 `compressor` 作为参数传入 `_save_conversation()`。
4. 压缩后的摘要存回 `conversation_store`，替换被压缩的消息段。

**验收**：构造 25 条消息的会话后，`conversation_store` 中早期消息被替换为一条摘要消息，新对话仍正常。

### Task 9 · SSE 接口改用 fetch + ReadableStream

**文件**：`frontend/app.js`（EventSource 相关代码）

**现状**：浏览器原生 `EventSource` 不支持自定义 header，token 只能放 query param，暴露在 URL 中。

**目标**：用 `fetch` + `ReadableStream` 替换 `EventSource`，Bearer token 放 Authorization header。

**改动步骤**：

1. 提取 SSE 消费逻辑为 `streamChat(query, sessionId, onEvent)` 函数：

```javascript
async function streamChat(query, sessionId, onEvent) {
    const resp = await fetch(`/api/chat/stream?query=${encodeURIComponent(query)}&session_id=${sessionId}`, {
        headers: { Authorization: `Bearer ${getToken()}` },
    });
    const reader = resp.body.getReader();
    const decoder = new TextDecoder();
    let buf = "";
    while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        buf += decoder.decode(value, { stream: true });
        const events = buf.split("\n\n");
        buf = events.pop();
        for (const raw of events) {
            const [eventLine, dataLine] = raw.split("\n");
            const type = eventLine.replace("event: ", "");
            const data = JSON.parse(dataLine.replace("data: ", ""));
            onEvent(type, data);
        }
    }
}
```

2. 原所有 `new EventSource(...)` 调用替换为 `streamChat(...)`。
3. URL 中去掉 `token` query param。

**验收**：打开浏览器网络面板，SSE 请求的 URL 不含 token；Authorization header 正确传递。

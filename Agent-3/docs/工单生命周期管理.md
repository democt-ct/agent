# 工单生命周期管理

> 版本 v1.0 · 2026-06-10  
> 对应实现: `src/skills/ticket_lifecycle.py`

---

## 1. 概述

工单生命周期模块在现有 `it_tickets` 表之上提供完整的工单状态机管理，
涵盖创建、处理、完成、关闭、重开、升级和统计七类操作。
所有操作均有严格的状态转移校验（FSM），非法转换会返回 error 而非静默失败。

## 2. 状态机

```
              ┌──────────────────────────┐
              │        待处理             │
              │   (初始状态)              │
              └─────┬──────────┬─────────┘
                    │          │
            process │          │ close
                    ▼          ▼
              ┌──────────┐  ┌──────────┐
              │  处理中   │  │  已关闭   │
              └────┬─────┘  └────┬─────┘
                   │             │
         complete  │             │ reopen
                   ▼             ▼
              ┌──────────┐  ┌──────────┐
              │  已完成   │  │  待处理   │
              │  (终态)   │  │  (重开)   │
              └──────────┘  └──────────┘
```

**合法转换矩阵**：

| 当前状态 → | 待处理 | 处理中 | 已完成 | 已关闭 |
|-----------|:-----:|:-----:|:-----:|:-----:|
| 待处理     |  —    |  ✅   |   ❌  |  ✅   |
| 处理中     |  ❌   |   —   |  ✅   |  ✅   |
| 已完成     |  ❌   |   ❌  |   —   |  ❌   |
| 已关闭     |  ✅   |   ❌  |   ❌  |   —   |

**升级**（escalate）不改变状态，仅将优先级设为「高」并通知部门经理。
任意非终态（待处理/处理中）均可升级。

## 3. API 参考

### 3.1 `create_ticket_full()`

创建工单，自动路由到部门经理并发送通知。

```python
create_ticket_full(
    user_id: str,           # 员工ID
    issue_type: str,        # 硬件报修/软件安装/网络问题/账号问题/设备申领/报销问题/合同问题/制度咨询/其他
    description: str,       # 问题描述
    priority: str = "中",   # 高/中/低
    department_id: str = "",# 留空则按类型自动路由
) -> dict
```

**返回示例**：
```json
{
  "success": true,
  "ticket_id": "TK004",
  "user_id": "EMP001",
  "status": "待处理",
  "assigned_to": "EMP005",
  "message": "工单 TK004 已创建，已分配给 EMP005，状态: 待处理"
}
```

**通知**：创建后自动通知受理人和申请人。

### 3.2 `process_ticket()`

开始处理工单：待处理 → 处理中。

```python
process_ticket(
    ticket_id: str,         # 工单ID
    operator_id: str,       # 操作人ID
    comment: str = "",      # 处理备注
) -> dict
```

### 3.3 `complete_ticket()`

完成工单：处理中 → 已完成。

```python
complete_ticket(
    ticket_id: str,
    operator_id: str,
    resolution: str = "",   # 解决方案描述
) -> dict
```

### 3.4 `close_ticket()`

关闭工单：待处理/处理中 → 已关闭。用于重复工单、用户撤销、无法处理。

```python
close_ticket(
    ticket_id: str,
    operator_id: str,
    reason: str = "",       # 关闭原因
) -> dict
```

### 3.5 `reopen_ticket()`

重开工单：已关闭 → 待处理。

```python
reopen_ticket(
    ticket_id: str,
    operator_id: str,
    reason: str = "",       # 重开原因
) -> dict
```

### 3.6 `escalate_ticket()`

升级工单优先级为「高」，通知部门经理。

```python
escalate_ticket(
    ticket_id: str,
    operator_id: str,
    reason: str = "",       # 升级原因
) -> dict
```

### 3.7 `get_ticket_detail()`

获取工单详情（含申请人姓名、部门、处理时长）。

### 3.8 `get_ticket_summary()`

工单统计摘要。

```python
get_ticket_summary(
    group_by: str = "status",  # status / type / priority / department
    days: int = 7,             # 统计最近 N 天
) -> dict
```

**返回示例**：
```json
{
  "total": 12,
  "group_by": "status",
  "groups": [
    {"label": "待处理", "count": 8},
    {"label": "处理中", "count": 3},
    {"label": "已完成", "count": 1}
  ],
  "by_priority": {"高": 2, "中": 8, "低": 2}
}
```

## 5. CLI 使用

```bash
# 创建
python scripts/ticket_ops.py create EMP001 硬件报修 "蓝屏无法启动" --priority 高

# 处理
python scripts/ticket_ops.py process TK004 EMP005 --comment "已确认故障"

# 完成
python scripts/ticket_ops.py complete TK004 EMP005 --resolution "更换主板"

# 关闭
python scripts/ticket_ops.py close TK005 EMP002 --reason "重复工单"

# 重开
python scripts/ticket_ops.py reopen TK005 EMP001 --reason "问题复现"

# 升级
python scripts/ticket_ops.py escalate TK003 EMP001 --reason "超 24h 未处理"

# 报表
python scripts/ticket_ops.py report --type daily
python scripts/ticket_ops.py report --type weekly --department dept-it

# 列表
python scripts/ticket_ops.py list --status 待处理 --priority 高

# 详情
python scripts/ticket_ops.py summary TK004
```

## 6. 工单类型路由

创建时按 `issue_type` 自动确定责任部门，进而路由到对应部门经理：

| issue_type | department_id | 受理人 |
|-----------|--------------|--------|
| 硬件报修 / 软件安装 / 网络问题 / 账号问题 / 设备申领 | `dept-it` | IT 部经理 (EMP005) |
| 报销问题 | `dept-finance` | 财务部经理 (EMP004) |
| 制度咨询 | `dept-hr` | HR 部经理 (EMP003) |
| 合同问题 / 其他 | `dept-it` | IT 部经理 (EMP005) |

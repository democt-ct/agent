## 🔄 日常 Git 操作（每次必做）

### 开工前：拉取最新代码

```powershell
cd D:\zhuomian\agent
git pull
```

⚠️ **不 pull 直接改 = 早晚冲突**，养成习惯。

### 收工前：保存并推送

```powershell
cd D:\zhuomian\agent
git add -A
git commit -m "一句话说清楚改了什么"
git push
```

### 完整的日常节奏

```powershell
# ===== 到工位/开机后 =====
cd D:\zhuomian\agent
git pull                    # 把另一台电脑的改动拉下来

# ... 写代码、改文件 ...

# ===== 走之前 =====
cd D:\zhuomian\agent
git add -A
git commit -m "改了什么"
git push                    # 推到 GitHub，另一台电脑 git pull 就能拿到
```

### 常用检查命令

```powershell
git status          # 哪些文件改了、新增了
git log --oneline   # 最近提交记录（按 q 退出）
git diff            # 具体改了什么内容
```

## 🚀 四个项目如何启动

### Agent-1 — API 服务（端口 8000）

旅游客服 AI Agent，FastAPI + ChromaDB 知识库。

```powershell
# 方式 1：用封装好的 bat
D:\zhuomian\agent\项目启动\启动Agent-1.bat

# 方式 2：手动启动
cd D:\zhuomian\agent\Agent-1
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

启动后访问：**http://127.0.0.1:8000**

---

### Agent-2 — 旅游助手（端口 9000 + 8787）

FastAPI 主服务 + 后台 Worker 双进程架构。

- FastAPI：**http://127.0.0.1:9000**

## 🐛 常见问题

### git pull 冲突了怎么办？

```powershell
git pull                          # 拉下来会提示冲突文件
# 手动编辑冲突文件，删掉 <<<<<< ====== >>>>>> 标记
git add -A
git commit -m "解决冲突"
git push
```

### git push 被拒（rejected）？

远程有别人（或另一台电脑）推了新内容，先 pull 再 push：

```powershell
git pull
git push
```

### 端口被占用？

看一下谁在占端口：
```powershell
netstat -ano | findstr :8080
```
找到 PID 后杀进程，或者用 `项目启动/` 目录下的 bat，它们会自动处理。

### .env 文件找不到？

`.gitignore` 排除了 `.env`，每台电脑需要自己创建。去另一台电脑把 `.env` 内容复制过来。
